# 桌宠 / 舰娘皮肤平级选择 — 设计

**日期**: 2026-07-19  
**状态**: 待实现  
**前置结论**: 方案 1（选择皮肤区顶栏双类）；问答 B（本期仍只播 Spine；独立「舰娘皮肤」功能关闭）

## 1. 目标

人物详情「选择皮肤」将 **桌宠** 与 **舰娘** 提升为平级分类（同一套皮肤槽、两种视图），整体交互与现网一致：点选切皮、桌宠窗播放。本期 **不改变** Spine 上桌行为；关闭「动作与台词 → 舰娘皮肤」整块（同步 / 预览 / 上桌）。

## 1.1 非目标（本期）

- 桌宠窗内播 Cubism（后续：舰娘类点选 → 同窗 Cubism）
- 恢复或增强独立舰娘预览窗、舰娘上桌入口
- 改皮肤包格式 / roster schema
- 设置页 `KanmusuSkinSettings` 全局同步（若存在则本期隐藏或保留只读提示，默认隐藏人物详情内入口即可）

## 2. 信息架构

```
人物详情
├── 选择皮肤
│   ├── [桌宠] [舰娘]     ← 平级 Tab（默认「桌宠」）
│   └── 皮肤卡片网格       ← 按当前 Tab 筛选同一 CharacterSkin 列表
└── 动作与台词
    ├── 动作分配
    ├── 台词
    └── 台词导入
    （移除「舰娘皮肤」Tab）
```

## 3. 筛选与点选规则

| 分类 | 列表条件 | 点选行为（本期） |
|------|----------|------------------|
| 桌宠 | `model_ready == true`（与现「可点 Spine」一致；未就绪仍可显示为 incomplete 卡） | `characters_set_skin(..., companion: "spine")` 或等价强制 Spine |
| 舰娘 | 有舰娘资源：`kanmusu_dir` 非空 **或** `kanmusu_ready` | 仍强制 Spine 切皮（`companion: "spine"`）；**不**调用 `companion: "kanmusu"` / 不上 Cubism |

说明：

- 同一 `skin.id` 可同时出现在两类（有小人又有舰娘资源时）；高亮态按「当前角色 active_skin_id」判断，与分类无关。
- 舰娘 Tab 为空：提示「当前角色暂无舰娘皮肤资源」，不提供同步按钮（同步入口关闭）。
- 卡片徽章：桌宠 Tab 可保留「小人」类提示；去掉「点卡片切小人 / 下方舰娘上桌」这类旧文案；舰娘 Tab 可不重复打「舰娘」徽标（分类已表达）。

## 4. 状态与 IPC

- **功能不变**：切皮、刷新、删除、导入皮肤仍走现有 IPC。
- **强制 Spine**：人物详情内皮肤点选一律传 `spine`（避免 `auto` 在已偏好 kanmusu 时误开 Cubism）。
- **companion 引擎**：若当前已是 kanmusu 引擎，切皮时仍写 spine 并切回桌宠播放器（与「二者都用当前桌宠播放器」一致）。
- **预留**：UI 记住上次打开的分类 Tab（session 或 localStorage 即可）；不必落库。将来舰娘类点选改为同窗 Cubism 时，只改点选分支，不改列表结构。

## 5. 关闭范围

从 `CharacterPetSettings` 移除：

- Tab「舰娘皮肤」
- 同步 / 预览 / 上桌按钮及关联 state

设置页若有「同步舰娘皮肤」入口：本期隐藏（或 disabled +「已迁入皮肤分类，播放器接入前暂关」）；默认以人物详情关闭为准。

帮助文案中「动作与台词 → 舰娘皮肤」改为「选择皮肤 → 舰娘（本期仍播桌宠）」类表述。

## 6. UI 要点

- 复用现有 `pet-tab` / `pet-model-card` 样式，皮肤区增加一层 kind Tab，避免新设计体系。
- 导入皮肤、分页仍挂在「选择皮肤」区块，对两类共用。
- 不显示内部 model / character id（沿用既有 UI 约束）。

## 7. 验收

1. 桌宠 Tab：点可选皮 → 桌宠窗加载对应 Spine；行为与改前一致。  
2. 舰娘 Tab：仅列出有舰娘资源的皮；点选后仍是 Spine，不出现 Cubism 上桌 / 独立预览。  
3. 「动作与台词」无「舰娘皮肤」Tab。  
4. 无舰娘资源角色：舰娘 Tab 为空态，不报错。  
5. 导入皮肤包后两类列表按资源就绪情况更新。
