# 桌宠动作表格、台词库与重启持久化

**日期**: 2026-07-03  
**主题**: 重启后随机动作失效、动作表 UI、台词、无 AI 模式

## 需求

1. 修复重启后桌宠/随机动作失效（元数据丢失或待机误选）
2. 动作列表改为表格：随机 / 待机 / 开机 / 回待机 / 点击
3. 测试环境存 AppData；打包时内置 `animations.meta.json` 随模型分发
4. 台词库：随机动作时说出；可导入；可 AI 一键添加；无 AI 时定时气泡用台词

## 实现

### 数据 `PetAnimationMeta`

新增字段：`boot_animation`、`return_idle_animation`、`lines[]`（`{ text, animation? }`）

持久化三层：
- SQLite `pet_anim_meta_{id}`
- `{data_dir}/pet-meta/{id}/animations.meta.json`（含内置柴郡）
- 打包默认 `public/assets/pet/chaijun/animations.meta.json`（Rust `include_str!`）

### 重启修复

- `normalize_idle_animation()`：拒绝非 idle 名（如 `dance`）
- `sync_animations` 不再接受无效 payload idle
- 冷启动无 DB 时合并 bundled 默认（含 random + lines）

### 前端

- `PetActionSettings.tsx`：动作分配表 + 台词库 + 保存
- `spinePet.ts`：开机/回待机动作；随机回调触发台词气泡
- `main.ts`：从 config.lines 选台词
- `mod.rs`：气泡优先级 AI → 台词库 → 时间线/本地

### IPC

- `pet_save_animation_layout`
- `pet_import_lines`
- `pet_ai_suggest_lines`

## 验证

1. 保存动作表 → 重启应用 → 随机仍播放、待机为 `normal`
2. 无 AI 配置时，定时气泡显示台词库内容
3. 随机动作播放时弹出绑定台词
