# 101 · 输入监控继续优化（Raw Input + 后台降优先级）

**日期**：2026-07-05  
**标签**：input_monitor、Raw Input、性能

## 背景

doc 100 移除 `WH_MOUSE_LL` 后仍保留 `WH_KEYBOARD_LL`，键盘低级别 Hook 仍可能拖慢输入；鼠标轮询精度一般。

## 优化

1. **全面移除低级别 Hook**，改用隐藏 `HWND_MESSAGE` + `RegisterRawInputDevices(RIDEV_INPUTSINK)` 接收 `WM_INPUT`。
   - 鼠标：仅处理 `RI_MOUSE_*_BUTTON_DOWN`，移动事件 `usButtonFlags == 0` 直接跳过。
   - 键盘：在消息线程内 `ToUnicode`，不在 Hook 链上执行。
2. **采集开关同步**：`set_tracking_enabled` → `set_input_enabled`，关闭采集时不计数。
3. **后台线程降优先级**：poller / input / audio / file_watcher 启动时 `THREAD_PRIORITY_BELOW_NORMAL`。
4. **文件监视**：`tracking_enabled == false` 时休眠，减少无效事件处理。
5. 消息等待间隔 50ms → **16ms**，退出响应更快。
