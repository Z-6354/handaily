# 87 · Wiki 人设导入 JSON 截断优化

**日期**：2026-07-04  
**类型**：Bug 修复 / 提示词优化

## 现象

Wiki 导入人设时出现：

```
JSON 不完整，正在压缩重试（3/3）…
AI 返回的 JSON 不完整（可能被截断）: expected `,` or `}` at line 1 column 149
```

思考模型（如 OpenCode GO 配额受限）输出 token 上限极低，JSON 预处理在约 150 字符处被截断。

## 根因

1. Wiki 清洗后的文本仍依赖思考模型做「参考文本 → JSON」预处理
2. 部分供应商思考通道输出极短或 JSON 落在 `reasoning_content` 且不完整
3. 原 compact 提示词仍可能诱导模型输出偏长 JSON

## 方案

### 1. Wiki 本地结构化（主路径）

`persona_builder::try_profile_from_wiki_reference` 从已清洗的 Wiki 参考文本直接解析：

- `# 角色：` / `来源：`
- `## 角色信息` 表格字段（性格、身份等）
- `## 角色设定` 等 prose 区块
- `## 舰船台词` 列表

解析成功则 **跳过 LLM JSON 预处理**，仅调用思考模型生成 Skill Markdown。

### 2. 提示词分层

| 文件 | 用途 |
|------|------|
| `persona-preprocess-minimal.md` | 极短 JSON 骨架，总输出 ≤600 字 |
| `persona-preprocess-compact.md` | 加强长度限制 + JSON 骨架 |
| `persona-skill-generate.md` | 缩短 Skill 篇幅 150～350 字 |
| `persona-text-merge.md` | 合并时也强调 JSON 闭合 |

Wiki 重试档位：`Minimal 4k → Minimal 1.5k → Compact 2.5k`。

### 3. API / 解析增强

- `ChatOptions.json_object`：OpenAI `response_format: json_object`、Ollama `format: json`
- JSON 任务专用 system：只输出合法 JSON，优先闭合
- `extract_openai_message`：content 与 reasoning 均含 `{` 时取较长者
- `parse_profile_json` 使用 `json_util::extract_json_object`

### 4. Wiki 提取再收紧

台词抽样 12 条、参考文本 ≤5500 字、单区块 ≤900 字。

## 使用建议

- **BWIKI 导入**：重启应用后重试；成功时会显示「已本地解析 Wiki 资料，跳过 JSON 预处理」
- **粘贴长文本**：若仍 JSON 截断，换 Ollama 本地思考模型或等待供应商配额重置
- 用户目录 `%AppData%/xiaohan-daily/prompts/` 若存在旧版 preprocess md，删除后重启可恢复新默认
