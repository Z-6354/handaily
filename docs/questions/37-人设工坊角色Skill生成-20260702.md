# 37 · 人设工坊：角色资料导入与 Skill 生成

**日期**：2026-07-02  
**类型**：新功能

## 需求

用户导入角色复合文本 → AI 轻量解析为结构化 JSON（介绍、台词等）→ 存 DB → 可 CRUD / 文本合并更新 → 由 JSON 生成 Skill 人设 Markdown → 预览编辑 → 写入 AI 人设系统。另新增**思考模型**槽位，专用于非对话、结构化文本任务。

## 实现

### 思考模型

- `ModelKind::Thinking`，配置项 `thinking_vendor_id` / `thinking_model`
- 设置 → AI 配置：第三栏「思考模型」
- `PreparedThinkingChat`：不注入当前人设，用于预处理 / Skill 生成
- 未配置思考模型时回退到文本模型

### 数据库 `character_profiles`

| 字段 | 说明 |
|------|------|
| slug / name / source | 标识与展示 |
| raw_text | 原始复合文本 |
| profile_json | 结构化资料（介绍、性格、台词等） |
| skill_md | 生成的人设 Markdown |
| persona_id | 写入 personas 后的 ID |

### 人设工坊流程（AI 人设页）

1. **新建角色**：粘贴大段文本
2. **AI 解析**：思考模型 → JSON（`persona-preprocess.md`）
3. **编辑 JSON**：手动改 / 「用文本更新资料」（`persona-text-merge.md`）
4. **生成 Skill**：`persona-skill-generate.md` → 预览编辑
5. **应用到 AI 人设**：写 `{id}.md` + 更新 `manifest.json`，可选设为当前

### IPC

`character_list/get/create/update_raw/update_json/save_skill/delete/preprocess/merge_text/generate_skill/apply_persona`

### 提示词

- `prompts/persona-preprocess.md`
- `prompts/persona-text-merge.md`
- `prompts/persona-skill-generate.md`

## 使用

1. 设置 → AI 配置 → 配置**思考模型**（建议选更强文本模型）
2. AI 人设 → 人设工坊 → 新建 → 解析 → 生成 Skill → 应用到 AI 人设
