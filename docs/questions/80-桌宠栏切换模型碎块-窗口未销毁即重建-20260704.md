# 80 — 桌宠栏切换模型碎块：窗口未销毁即重建

**日期**: 2026-07-04  
**主题**: 下拉切换模型碎块；关闭桌宠再启用正常

## 现象

桌宠页「当前模型」下拉切换后 Spine 部件碎块；关闭桌宠再启用同一模型则正常。

## 根因

两条路径设计目标一致（重建 WebView），但时序不同：

| 路径 | 行为 |
|------|------|
| **关闭再启用** | `hide_pet(destroy:true)` → 用户操作间隔 → `show_pet` → 新 WebView |
| **栏内切换（修复前）** | `restart_pet_window` → `win.close()` → 立即 `ensure_pet_window` |

`win.close()` 在 Windows 上异步；`ensure_pet_window` 若仍检测到 `PET_LABEL` 窗口存在则 **跳过创建**，在旧 WebView 上发 `pet-reload`，同 canvas dispose/重建 → 碎块（与 doc 79 同类，但入口是模型切换而非预览）。

## 修复（`hanpet/src-tauri/src/pet/mod.rs`）

1. `destroy_pet_window`：`close()` 后轮询直至窗口从 Tauri 注销（最长 2s），并重置 `position_guard_attached`。
2. `restart_pet_window`：销毁前 `companion_session_stop`；去掉固定 80ms sleep，改由上述等待保证 GL/WebView 释放后再 `show_pet` / `ensure_pet_window`。

`set_active_model` 在桌宠启用时仍走 `restart_pet_window`，与「关闭再启用」同路径。

## 验证

重新编译并重启应用；桌宠可见时在模型间多次切换，应无碎块（可能有短暂窗口闪烁，属预期）。
