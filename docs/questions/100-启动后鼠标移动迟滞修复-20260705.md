# 100 · 启动后鼠标移动迟滞修复

**日期**：2026-07-05  
**标签**：input_monitor、Hook、性能

## 现象

应用启动后，系统鼠标移动明显迟滞、不跟手。

## 根因

`input_monitor.rs` 安装了 **`WH_MOUSE_LL` 低级别鼠标 Hook**。该 Hook 会在**每一次鼠标移动**时同步进入回调，即使只统计点击，也会拖慢全系统光标响应。此前为修复退出卡住改用 `PeekMessageW` + 50ms sleep，但鼠标迟滞的主因仍是 LL 鼠标 Hook。

键盘 Hook 内在 `ToUnicode` + 双 Mutex 也会加重输入延迟，属次要因素。

## 修复

1. **移除 `WH_MOUSE_LL`**，改为 `GetAsyncKeyState` 轮询左/中/右键**按下边沿**统计点击（不拦截移动事件）。
2. **键盘 Hook 轻量化**：回调内只做计数 + `try_lock` 入队 `(vk, scan)`；`ToUnicode` 改在消息泵线程 `drain_key_queue` 中执行。
3. 消息等待改用 `MsgWaitForMultipleObjects(..., QS_ALLINPUT)`，有输入时立即唤醒，退出仍可在约 50ms 内响应 `stop_flag`。

## 涉及文件

- `src-tauri/src/tracker/input_monitor.rs`

## 说明

鼠标点击统计改为边沿轮询，极短连点（< 一帧轮询间隔）可能漏计，对日报级指标可接受。
