# 109 · 时间线 AI 切换页面与无密钥场景审计

**日期**：2026-07-05  
**标签**：时间线、AI、页面切换、密码本、Bug 修复

## 用户问题

切换页面（含后台）时时间线 AI 是否正常？未配置 AI 密钥时功能是否完善？

## 审计结论

### 后台时间线 AI（Rust `timeline/scheduler.rs`）

- 应用启动约 12s 后开始，每 90s 补全**今日未缓存**片段，**不依赖**是否打开时间线页。
- 采集暂停时不运行；结果写入 SQLite cache + JSON 日志。
- 主窗口隐藏不影响后台任务。

### 切换页面行为

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 离开时间线页 | 组件卸载，事件监听移除；后台仍写 cache，再进入 `load()` 读 cache | 同左（设计合理） |
| 留在时间线、后台补全 | `applyChunk` 要求 offset/limit 完全一致，**后台 emit 的 limit=全量条数，前端 limit=50 → 增量事件被丢弃** | 按当前页 `started_at` 匹配合并 chunk |
| 切页时进行中的 `timelineDescribe` | 后端继续完成并落库，再进入可读到 | 不变 |

### 无 AI / 无密钥（修复前缺陷）

1. **`aiIsTextReady=false` 时直接 return**，不读 cache、不调 `timelineDescribe`。
2. 后端仍会生成**本地简介**（`used_ai=false`）并缓存，前端却只用 `fallbackText`，**工作类型标签也丢失**。
3. 提示语笼统「未绑定 AI」，未区分：未选模型 / 未关联密钥 / **密码本未解锁** / Ollama 未就绪。

### 修复（`TimelineView.tsx`）

1. 无论 AI 是否就绪：读 `timelineCached` + 对未缓存项调 `timelineDescribe`（后端无密钥时走本地 fallback 并落库）。
2. `applyChunk`：只合并与**当前列表** `started_at` 匹配的 entries（兼容后台调度分页参数）。
3. `resolveAiHint()`：按配置/密码本状态给出可操作提示。

### 设置页 AI 密钥（已有能力）

- 「设置 → AI 配置」：供应商关联密码本条目、测试连接、选模型。
- Ollama 无需密钥；云端供应商需密码本解锁 + `vault_entry_id`。
- 密码本独立页「更多 → 密码本」。

## 手动验证建议

1. 未配 AI：打开时间线 → 应见具体提示 + 本地简介/活动记录。
2. 配好密钥但未解锁密码本 → 提示「密码本未解锁…」。
3. 打开时间线后切到其他页，等 1～2 分钟再回来 → 简介应已更新（cache 或 chunk）。
4. 仅托盘/桌宠、主窗隐藏：后台仍补全，再开主窗进时间线应看到 cache。
