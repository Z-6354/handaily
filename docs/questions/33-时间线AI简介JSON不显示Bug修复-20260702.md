# 33 · 时间线 AI 简介/JSON 不显示不落盘 Bug 修复

**日期**：2026-07-02  
**类型**：Bug 修复

## 现象

工作时间线 AI 简介不显示，`timeline-ai/` 下无 JSON 文件。

## 根因

1. **`PreparedTextChat::prepare()?`**：密码本未解锁、缺 API Key 等会直接 **Err**，整段 `timeline_describe` 失败 → 无简介、无 JSON、无落盘
2. **`save_batch_log` 错误被 `let _ =` 吞掉**，写文件失败无日志
3. 文件名时间戳 `%H%M%S%.3f` 在部分环境可能异常，改为 `HHMMSS-mmm`
4. 返回结果未统一从 DB 重读，可能与缓存不一致

## 修复

- `prepare` 失败 → 走本地简介 + 仍写 JSON（含 `response.error`）+ 仍 `persist_entries`
- 写 JSON 失败/成功 `eprintln!` 到控制台
- `app_get_timeline_ai_logs_path` 调用时 `ensure_logs_dir`
- 结束时 `get_cached` 全量 keys 再返回
- 前端：简介错误单独提示；日志目录加载失败时用 `data/xiaohan.sqlite` 推导路径

## 验证

1. **完全重启**应用（`npm run tauri dev`）
2. 打开工作时间线，看底部 JSON 目录路径
3. 查看 `%AppData%/xiaohan-daily/data/timeline-ai/{日期}/`
4. 若仍本地简介：检查设置中文本模型、密码本是否解锁
