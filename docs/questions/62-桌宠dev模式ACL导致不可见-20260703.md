# 桌宠 dev 模式 ACL 导致透明不可见

## 问题

用户已开启桌宠且配置正常，仍完全看不到桌宠窗口。

## 根因

1. **开发模式 URL 错误**：`pet_url()` 在 `debug_assertions` 下使用 `WebviewUrl::External("http://localhost:1420/pet.html")`。Tauri 2 将该地址视为**远程源**，对 `pet_get_config` 等 invoke 执行 ACL；未授权时 IPC 失败，Spine/静态图均无法加载 → **透明空窗**。
2. **主窗口无此问题**：`main` 使用 `tauri.conf` 默认 `index.html`（App 协议 / 本地源），ACL 不拦截应用 command。
3. **次要因素**：`setup()` 同步 `WebviewWindowBuilder` 在 Windows WebView2 上可能不稳定；桌宠未写入 `tauri.conf` 时仅靠运行时创建。

## 修复

| 项 | 改动 |
|----|------|
| URL | `pet_url()` 统一 `WebviewUrl::App("pet.html")`，dev 由 Tauri 走 devUrl |
| 窗口 | `tauri.conf.json` 预注册 `label: pet` 透明置顶窗（初始 `visible: false`） |
| ACL | `permissions/pet-ipc.toml` + `pet-capability` 增加 `allow-pet-ipc` |
| 启动 | `sync_on_startup` 全程异步重试 `show_pet` + `pet-reload` |
| 诊断 | `pet/main.ts` IPC 失败时显示 `.pet-load-error` 条幅 |

## 相关文件

- `hanpet/src-tauri/src/pet/mod.rs`
- `hanpet/src-tauri/tauri.conf.json`
- `hanpet/src-tauri/capabilities/pet.json`
- `hanpet/src-tauri/permissions/pet-ipc.toml`
- `hanpet/src/pet/main.ts`、`pet.css`

## 验证

`npm run tauri:dev` 启动后约 1s 内右下角应出现柴郡；若 IPC 仍失败会看到红色错误条幅而非完全透明。
