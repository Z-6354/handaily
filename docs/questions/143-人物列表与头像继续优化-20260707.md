# 143 · 人物列表与头像继续优化

**日期**：2026-07-07

## 背景

在 141/142 分页与内存优化基础上，列表翻页仍会因头像批量下载阻塞 loading、每人 4 次 stat 查本地头像、单组件与 batch 重复下载、桌宠预加载整包 PNG 占用内存。

## 改动

### Rust

- **`build_avatar_path_index`**：列表页一次 `read_dir` 解析本页 `avatar_path`，替代逐人 `stat` 多扩展名
- **`ensure_avatars_cached_batch`**：4 路 `tokio::spawn` 并发下载；复用全局 `reqwest::Client`
- **`run_avatar_import_default`**：导入同样 4 路并发
- **`ResolvedRosterCache`**：缓存 `model_ids`，`load_resolved_roster` 不再二次扫描模型目录

### 前端

- **`useCharacterRoster`**：先 `setLoading(false)` 展示列表，再后台 `charactersCacheAvatarsBatch`
- **`CharacterAvatar`**：`deferDownload` 列表卡片开启，避免与 batch 重复 IPC
- **`pet/main.ts`**：预加载仅 skel/atlas/config，PNG 按需 `urlFor` 读取

## 使用说明

- 列表先出文字与占位首字，头像在后台陆续补齐
- 详情页仍单张 `characters_cache_avatar` 自动下载
- 批量导入：`npm run avatar:import -- --limit 50`
