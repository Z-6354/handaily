# 审计高优项修复

## 问题

按 `docs/questions/54-项目隐患与优化审计-20260703.md` 建议，修复 H1–H6 与 M2。

## 修复摘要

### H4 — 时间线懒加载
- `TimelineView` 仅在 `active === true` 时首次 `load(0)`，避免启动即跑 AI

### H1 — 截图分析释放 DB 锁
- `vision.rs` 拆分 `prepare_screenshot`（持锁）与 `execute_prepared`（网络 I/O）
- `coordinator.rs` 先准备再执行，不再持锁调 AI

### H2 — 聚合器与 DB 一致
- `writer.rs`：`insert_segment` 成功后才 `apply_to_aggregator` + 入分析队列
- 插入失败打日志并跳过

### H3 — 午夜整点时段
- `period_scheduler.rs`：`00:00` 时 `date_naive` 回退到昨天再分析 23 点时段

### H5 — 桌宠滑块 debounce
- `PetPanel.tsx` 角色大小 350ms debounce 后再保存/reload

### H6 — 全局 refresh 竞态
- `App.tsx` 增加 `refreshGenRef`，丢弃过期响应
- 移除无用的 `getTodayMetrics` 轮询

### M2 — timeline_describe 并发锁
- `AppState.timeline_describe_lock`（`tokio::sync::Mutex`）
- `timeline_describe` 全程持锁，防重复 AI 批次

### 附加
- `pet/main.ts`：`reloadPet` 串行化，防并发 Spine 初始化
