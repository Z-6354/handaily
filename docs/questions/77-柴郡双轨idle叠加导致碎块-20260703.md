# 柴郡预览仍碎块：track 1 重复 idle 双轨叠加骨骼

**日期**: 2026-07-03  
**主题**: 76 修复后用户仍反馈预览 dance/touch 碎块

## 根因（脚本验证 chaijun.skel）

1. **track 1 先播 idle 再 mix dance（76 方案）**  
   track 0 与 track 1 **同时**应用 `normal` 骨骼时间轴 → 首帧后约 51/68 根骨骼偏离已组装姿态 → 部件飞散。  
   `@0` 看似正常，**`update(1/60)` 后立即碎块**。

2. **track 1 直接 dance（无 idle 种子）**  
   骨骼会被 dance 第 0 帧重写；需 **锁定 idle 组装骨骼**，并只修复被 `apply()` 末尾「未 key → setup 附件」打散的 slot。

3. **柴郡动作特征**  
   `dance`/`touch` 等在 t=0 有 9 条 attachment 时间轴 + 204 条骨骼时间轴；`normal` 的 attachment 首 key 在 ~2.7s。

## 修复（`spinePet.ts`）

- **删除** track 1 idle 种子 + `addAnimation` 链
- track 0：永远循环 `normal`
- track 1：`setAnimation(ACTION_TRACK, dance, false)`，`mixDuration=0`
- `restoreAssembledPoseGuard`：**全量恢复骨骼**到 snapshot；附件仅当 slot 被重置为 setup 默认附件时才恢复
- 快照增加 `assembledSlotAttachments`（Attachment 引用）
- `patchSpineUpdate`：autoUpdate 路径在 `update()` 末尾再锁一次

## 验证

重启 dev / 立即更新 → 预览 dance/touch 不应碎块；骨骼保持站立，附件变化仍可体现部分动作效果。

## 局限

骨骼不随 dance 大幅变形（模型 bind pose 与动作设计限制）；若需完整舞蹈需换用 bind 即组装的 Spine 资源或改导出。
