# 编辑范围修复 — 已证错误方案清单

**日期**: 2026-07-11  
**状态**: 上述方案均已回滚或替换，勿再使用

## 背景

多轮 patch 后仍出现：拖任意边时模型/对边一起动；多次编辑后模型无法拖移。根因是**在错误层级做补偿**（改 `stageOffset` / 不同步 canvas / 混用坐标系），而非统一 resize 模型。

---

## 错误方案（禁止复用）

| # | 方案 | 为何错误 |
|---|------|----------|
| E1 | 拖拽预览**只动 OS 窗口、不同步 canvas** | stage 随窗口变大，canvas 仍居中/贴底 → 模型在 flex 布局里漂移 |
| E2 | 松手/提交时用 `compensateStageOffsetForResize` 改 `stageOffsetX/Y` | `stageOffset` 是用户拖移意图；resize 多次累加导致偏移爆炸，`clampStageOffset` 锁死 → **模型无法再拖** |
| E3 | 拖拽中 `syncCanvas` + E2 补偿 | 双重调整 + 与 `transform-origin: bottom center` 耦合，各边仍漂 |
| E4 | 仅用 Tauri `outerPosition/outerSize` 读写，不用 frame `GetWindowRect` | WebView frame 与 Tauri 外框不一致，锚点算错（部分正确，但单独不够） |
| E5 | 仅用 Rust `moveX/moveY` + 锚点公式，不改 canvas/Spine | OS 窗口对了，Web 层 canvas/Spine 未跟随 → 视觉仍错 |
| E6 | 退出/进入编辑时 `syncCanvasFromWindow(true)` refit | `fitSpineToCanvas` 重排模型，与用户 offset/scale 叠加 → **退出后跳变** |
| E7 | 编辑模式 overlay `pointer-events: auto` 盖满 stage | 挡住角色偏移拖动 |
| E8 | 进入编辑前菜单关 → 立即 `startClickThrough()` | 编辑模式收不到鼠标事件 |
| E9 | resize 期间 `setRenderPaused(true)` | 假「加载中」卡顿感 |
| E10 | `lockCanvasDisplaySize` 锁死 canvas | 预览尺寸与窗口脱节 |

---

## 正确模型（2026-07-11 起）

1. **窗口**：frame HWND + `SetWindowPos`，`moveX/moveY` 固定对边（Rust 保留）
2. **Canvas**：编辑中始终与窗口同尺寸（100% / px 同步）
3. **Spine**：`resizeCanvasForEditResize` — 北/西边 resize 时在 **canvas 内**平移 Spine（按 `stageScale` 换算），**不 refit**
4. **stageOffset**：仅用户拖移/滚轮缩放时使用；**resize 期间禁止写入**
5. **transform-origin**：编辑模式 `0 0`，正常模式 `bottom center`
6. **进入编辑**：从 DB 重载 layout（offset/scale），清除 E2 造成的脏偏移

## 验证矩阵

| 场景 | 预期 |
|------|------|
| 拖 N/S/E/W 单边 | 仅动该边，模型屏幕位置不变 |
| 编辑中拖角色 | offset 变化，松手后仍可再拖 |
| 多次进入/退出编辑 | offset 不累积漂移 |
| Esc / 点桌面退出 | 保存 layout，模型不跳 |
