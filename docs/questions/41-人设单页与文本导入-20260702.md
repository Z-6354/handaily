# 41 · 人设页单页布局与文本导入

**日期**：2026-07-02  
**类型**：UI / 人设

## 需求

- 去掉左侧「选择人设 / 人设详情 / 导入文件」标签
- 所有人设卡片同页展示
- 卡片列表末尾增加「新增人设」框，支持粘贴文本创建
- 详情页也支持文本导入更新

## 实现

### 布局

- **主列表**：人设卡片网格 + 底部通栏「新增人设」
- **详情**：点击「查看详情」进入全页详情（← 返回），底部「导入文本」区

### 文本导入

- 组件 `PersonaTextImportForm`（创建 / 更新两种模式）
- 后端 `persona_import_text`：
  - 以 `{` 开头 → 解析 JSON 写入结构化资料
  - 其它 → 作为 Skill Markdown 保存
- 仍支持「选择文件」（.md / .json / .txt），走原有 `persona_import`

### 新增人设

- 需填写人设 ID（必填）、显示名称（可选）
- 粘贴文本或选文件后点「创建人设」

## 使用

1. 主列表选用角色，或在「新增人设」粘贴内容创建  
2. 详情页底部可粘贴文本更新该角色的 Skill 或 JSON 资料

## 更新（2026-07-02）

详情页「导入文本」改为与 Skill 文档 / 结构化资料 / JSON **同级 Tab**，不再单独置于底部。
