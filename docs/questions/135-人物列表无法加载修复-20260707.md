# 135 · 人物列表无法加载修复

**日期**：2026-07-07

## 现象

人物页一直显示「加载人物…」或列表为空。

## 根因

1. **全屏 loading 阻塞**：每次 `refresh()`（含窗口 focus）都 `setLoading(true)`，整页被 loading 覆盖；focus 频繁触发时像永远加载不完
2. **IPC 参数**：向 `characters_list_page` 传 `null` 可选字段，部分 Tauri 版本反序列化失败
3. **无降级**：分页 IPC 失败后没有回退到 `charactersListBrief` / `charactersList`
4. **重复读盘**：`list_characters_page` 内 `resolve_all_characters` + `load_manifest` 各读一遍 manifest

## 修复

- 仅首次无数据时全屏 loading；刷新时保留列表并显示「刷新中…」
- focus 刷新节流 30s
- IPC 只传有值字段；失败时降级 brief → full list
- `load_resolved_roster` 单次合并 manifest
- 移除 `lib.rs` / permissions 中重复的 `live2d_import_batch`

## 验证

```powershell
cd hanpet/src-tauri
cargo run --bin roster_check
# 应输出 total: 866  items: 48
```

重启应用后人物页应显示「共 866 位 · 已加载 48」。
