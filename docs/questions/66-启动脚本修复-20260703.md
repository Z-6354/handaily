# 启动脚本修复

## 问题

`scripts/start-dev.ps1` / `npm run tauri:dev` 启动失败或编译报错。

## 根因

1. **Rust 编译错误**：桌宠 dev 改动引入 `use url::Url`，但 `Cargo.toml` 未声明 `url` 依赖 → `cargo` 无法完成，启动脚本表现为失败。
2. **RC.EXE 未在 PATH**：Windows 下 Tauri 构建需要资源编译器；普通 PowerShell 未加载 VS 环境时会报 `RC.EXE not in PATH`。
3. **预览配置路径**：`tauri.preview.conf.json` 放在仓库根目录，与 Tauri 项目目录 `src-tauri/` 不一致，改为 `--config src-tauri/tauri.preview.conf.json` 并移入该目录。

## 修复

| 项 | 改动 |
|----|------|
| Rust | `pet/mod.rs` 改用 `tauri::Url`，移除未声明的 `url` crate |
| `_common.ps1` | 新增 `Initialize-MsvcEnvironment`（Launch-VsDevShell / Windows SDK rc.exe） |
| `start-dev.ps1` / `build.ps1` | 启动前加载 MSVC 环境；更新提示文案 |
| `Invoke-TauriDev` | exit 101 时提示 RC.EXE / exe 占用 |
| `package.json` | `tauri:dev` 指向 `src-tauri/tauri.preview.conf.json` |
| 配置 | `tauri.preview.conf.json` 移至 `src-tauri/` |

## 使用

```bat
scripts\start-dev.bat
```

或 `npm run tauri:dev`（推荐仍通过 bat/ps1，会自动清理端口与加载 VS 环境）。

若仍报 RC.EXE：安装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)，勾选「使用 C++ 的桌面开发」。
