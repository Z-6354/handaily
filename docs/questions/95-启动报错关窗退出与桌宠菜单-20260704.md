# 启动报错、关窗退出与桌宠右键菜单

**日期**：2026-07-04  
**分类**：稳定性 / 桌宠

## 现象

1. 桌面安装后启动出现「系统找不到指定的注册表项或值」及 vision AI 配额报错
2. 关闭某页面后整个应用退出（托盘也消失）
3. 桌宠右键菜单缺少切换模型/性格，点击外部无法关菜单
4. 希望同路径重装为更新而非并行安装

## 根因与修复

### 注册表报错

`autostart_enabled=0` 时仍执行 `reg delete`，键不存在则 reg.exe 输出中文错误。  
→ 先 `reg query` 确认存在再删除。

### 关窗误退出

`RunEvent::ExitRequested` 在关主窗口时 `code=None`，此前仍设 `stop_flag` 且未 `prevent_exit()`。  
→ `code=None` 时 `api.prevent_exit()` 且不设 stop_flag；仅 `app.exit(0)` 等带 code 时真正退出。

### Vision 配额

后台截图分析触发 API 配额超限，stderr 打印整段 JSON。  
→ 识别 quota/rate limit 后静默降级为 fallback，仅一行简短日志。

### 桌宠菜单

- 新增「切换模型」「切换性格」子列表（`pet_list_models` / `persona_list`）
- `mousedown` 点菜单外、窗口失焦时关闭菜单
- 错误提示条可点 × 关闭，不会触发退出

### 安装包更新

NSIS：`installMode: currentUser`、`allowDowngrades: false`、简体中文；同用户重装覆盖旧版。
