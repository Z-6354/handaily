# 桌宠不可见：屏幕外坐标

## 问题

ACL / URL 修复后桌宠仍不可见。

## 根因

查用户 DB `app_settings`：

```
pet_x=-160
pet_y=-200
pet_width=160
pet_height=200
pet_enabled=1
```

窗口被定位到 **(-160, -200)**，完全在可见区域外（负坐标）。

`show_pet` 旧逻辑仅当 `x/y > 4000` 才重置，**负坐标与屏外正坐标均不会纠正** → `win.show()` 成功但用户看不到。

坐标可能来自拖拽保存时 `outerPosition` 与 DPI/多显示器换算偏差。

## 修复

- `virtual_screen_bounds` / `is_position_visible` / `clamp_pet_position` / `resolve_pet_position`
- `show_pet`：屏外则重置到右下角并写回 DB
- `save_position`：保存前 clamp 到虚拟桌面内
- `screen_bottom_right`：改用虚拟屏幕 metrics（多显示器）
- `pet.json`：补充 dev `remote.urls`（localhost / 127.0.0.1）作 ACL 兜底

## 验证

重启后 stderr 应出现：`桌宠: 位置 (-160,-200) 在屏幕外，已重置为 (...)`，右下角可见桌宠。
