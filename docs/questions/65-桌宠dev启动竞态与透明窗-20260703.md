# 桌宠依旧不可见（dev 启动竞态）

## 问题

ACL、全屏误判等修复后，用户仍完全看不到桌宠。

## 根因

1. **tauri.conf 预创建 pet 窗口**：应用启动时立即以 `App(pet.html)` 加载，与 `npm run dev` 启动 Vite **竞态**；首屏加载失败 → WebView 空白 + 窗口透明 → 肉眼不可见。
2. **仅 800ms 延迟不足**：`sync_on_startup` 未确认 Vite 已监听 1420 即 `show()`，`pet-reload` 事件可能在 JS 未执行时发出。
3. **Windows 透明 WebView2**：未显式 `set_background_color(0,0,0,0)` / `set_always_on_top`，部分环境透明窗不可见。
4. **dev 下 App 协议 secondary 窗口**：不如直连 `http://127.0.0.1:1420/pet.html` 可靠（pet 窗口 capability 已含 `allow-app-commands`，invoke 可用）。

## 修复

| 项 | 改动 |
|----|------|
| 窗口创建 | 从 `tauri.conf.json` **移除**预注册 pet 窗；改为 Vite 就绪后 `WebviewWindowBuilder` 动态创建 |
| 启动时序 | `wait_frontend_ready`（reqwest 探测 `/pet.html`）→ `show_pet` → 600ms → `nudge_pet` |
| dev URL | `pet_webview_url()`：debug 用 `External(127.0.0.1:1420/pet.html)`，release 用 `App(pet.html)` |
| 显示 | `prepare_pet_webview`：透明背景 + alwaysOnTop + navigate/reload |
| 加载完成 | `on_page_load(Finished)` 自动 emit `pet-reload` |
| 前端诊断 | 加载前显示粉色「桌宠加载中…」条幅，避免纯透明 |

## 验证

1. `npm run tauri:dev` 完全重启（非热重载 Rust）
2. 约 2–5s 后屏幕右下角应出现粉色「桌宠加载中…」或柴郡
3. 若只见条幅不见模型 → 看是否出现红色 IPC 错误提示
4. 设置 → 桌宠 → 确认「启用桌宠」开启，可点「显示桌宠」
