# 122 · 构建统一为 release-fast 单配置

**日期**：2026-07-06  
**标签**：构建、Cargo、简化

## 需求

编译方式只保留一种发布配置（原 `release-fast`），去掉 `release`（LTO）、`release-small` 及多套 npm 脚本。

## 改动

| 项 | 变更 |
|----|------|
| `src-tauri/Cargo.toml` | `[profile.release]` 直接采用 fast 参数（`lto=false`、`codegen-units=256`）；删除 `release-fast`、`release-small` |
| `package.json` | 移除 `tauri:build:full`、`tauri:build:fast`、`tauri:build:small`；`tauri:build` / `tauri:build:exe` 不再传 `--profile` |
| `scripts/build.ps1` | 移除 `-Full`、`-Small`；新增 `-TypeCheck`（等价 `FE_BUILD_FULL=1`） |
| `.cargo/config.toml` | 仅保留 `[profile.release]` incremental |

## 命令速查

```bash
npm run tauri:build          # NSIS + exe（release-fast 参数）
npm run tauri:build:exe      # 仅 exe
scripts\build.ps1 -ExeOnly
scripts\build.ps1 -TypeCheck # 打包前 tsc + vite
```

## 产物路径

```
src-tauri/target/release/xiaohan-daily.exe
src-tauri/target/release/bundle/nsis/小寒日报_0.1.0_x64-setup.exe
```

> 旧版 `target/release-fast/` 为历史产物，可删除；新构建统一输出到 `target/release/`。

## 说明

Cargo 默认发布 profile 名为 `release`，无法改名。将 fast 参数写入 `[profile.release]` 后，无需再 `--profile release-fast`，行为与此前 fast 模式一致。
