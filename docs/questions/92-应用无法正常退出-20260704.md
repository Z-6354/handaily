# 应用无法正常退出

**日期**：2026-07-04  
**分类**：tracker / 退出

## 现象

托盘或桌宠右键菜单点击「退出」后进程不结束，或长时间卡住。

## 根因

1. **键鼠 Hook 线程阻塞**：`input_monitor.rs` 用 `GetMessageW` 跑消息泵；该调用会无限阻塞，直到有新消息。退出时 `stop_flag` 虽已置位，但线程若正卡在 `GetMessageW` 内，无法检查标志。
2. **`RunEvent::Exit` 中 `join_all`**：会等待 poller / input / file / audio 四个后台线程结束；input 线程不退出 → `join` 永久挂起 → 进程无法退出。
3. **次要**：`app_exit` / 托盘 quit 未提前设 `stop_flag`、未销毁桌宠窗口，退出响应偏慢。

## 修复

**`hanpet/src-tauri/src/tracker/input_monitor.rs`**

- 将 `GetMessageW` 改为 `PeekMessageW` + `PM_REMOVE` + 50ms `sleep` 轮询
- 收到 `WM_QUIT` 时跳出循环，再 `UnhookWindowsHookEx`

**`hanpet/src-tauri/src/ipc/commands.rs` / `hanpet/src-tauri/src/tray.rs`**

- 退出前：`stop_flag = true` → `destroy_pet_window` → `app.exit(0)`

## 说明

主窗口点关闭按钮只会隐藏到托盘（设计如此），需用托盘或桌宠菜单「退出」才能真正结束进程。

## 验证

重新编译后：托盘/桌宠「退出」应在 1 秒内结束进程。
