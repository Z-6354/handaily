# 桌宠显示层重构：删除碎块组装路径

**日期**：2026-07-05  
**分类**：桌宠 / 稳定性

## 问题

此前为缓解 Spine 碎块，引入了 `pumpAnimationState` 预推进、`assembleBaseIdle`、`finalizeVisibleAssembly`、`finalizePetAssembly`、`refitPetAfterVisible` 等「组装」路径。这些逻辑在 hidden/visible、随机动作、reload 竞态下仍会强行改 track，反而成为碎块来源。

## 方案

**只保留正常 Spine 显示**：`autoUpdate = true` + 单轨 `setAnimation` / `addAnimation`，不再手动 pump 骨架。

### 删除

| 项 | 说明 |
|----|------|
| `pumpAnimationState` | `viewerExApply.ts` 导出及全部调用 |
| `assembleBaseIdle` / `finalizeVisibleAssembly` | `spinePet.ts` |
| `finalizePetAssembly` / `refitPetAfterVisible` | `main.ts` |
| `PowerMode` / `minimal` 分支 | 后端已恒为 balanced，前端不再保留静态图快路径 |
| `setPowerMode` | 构造函数不再接收 powerMode |

### 保留并简化

- `ensureIdleLoop()`：仅在非动作播放时 `setAnimation(idle, true)` + `render`
- `resumePetFromHidden()`：恢复 `autoUpdate` + `resizeCanvas` refit，不碰动画 track
- 首启 / 切换模型仍走 `pet-reload`；隐藏后再显示走 `pet-resume`（doc 117）

## 相关文件

- `hanpet/src/pet/spinePet.ts`
- `hanpet/src/pet/main.ts`
- `hanpet/src/pet/viewerExApply.ts`

## 验证

1. 冷启动桌宠：待机正常，无碎块
2. 隐藏再显示、全屏切换：不 reload，姿态稳定
3. 点击 / 随机动作 / 拖拽：动作结束后回到 idle
4. 切换模型：全量 reload 后正常
