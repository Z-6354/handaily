# 13 · 模型导入与火山引擎测试修复

**日期**：2026-07-02  
**类型**：Bug 修复 + 功能增强

## 问题

1. 「导入模型」仅返回内置硬编码列表，未从供应商 API 拉取  
2. 火山引擎测试报错 `API 返回错误:`（空 body）——URL 被错误拼成 `/api/v3/v1/models`

## 根因

`models_list_url` 对已是 `/api/v3` 的 base 仍追加 `/v1/models`，火山方舟正确路径为：

`GET https://ark.cn-beijing.volces.com/api/v3/models`

DeepSeek 等无版本段的 base 才需 `/v1/models`。

## 修复

### `hanpet/src-tauri/src/ai/urls.rs`（新增）

统一 `chat_completions_url` / `models_list_url`：
- `…/api/v3` → `…/api/v3/models`
- `https://api.deepseek.com` → `…/v1/models`

### `providers.rs`

- OpenAI 兼容供应商：`GET /models` + Bearer 密钥，解析 `data[]`  
- 导入/测试均走真实 API；Ollama 仍用 `/api/tags`  
- 火山 404/空体时提示开启 **Model Management** + **Chat API** 权限  
- 多模态导入按 id 关键词过滤 vision 模型

### `ai_import_models`

- 导入结果写入 `AiConfig.imported_models` 并持久化  
- `models_for` 合并：自定义 → 已导入 → 内置（去重）

## 用户操作（火山方舟）

若测试仍失败，请到火山方舟控制台 → API 密钥 → 编辑权限，勾选：
- Chat API  
- Model Management
