# 34 · 时间线 AI 上下文丰富注入

**日期**：2026-07-02  
**类型**：功能增强

## 需求

提交给时间线 AI 的原始片段过少。需按应用类型从窗口标题等提取更多结构化上下文，例如：

- **Cursor**：打开文件、项目名、会话/提问标题
- **Edge/浏览器**：网页标题、站点线索、标题内 URL
- **VS Code / 终端 / 微信 / QQ**：对应字段与可读 hints

并合并时段内混合分析洞察、同应用近期洞察。

## 实现

### 1. `tracker/context_enrich.rs`

- `SegmentEnrichment`：`app_kind`、`fields`、`hints`
- `enrich_segment(seg)` 按 `aggregation_key` / 应用名识别类型并解析标题
- 单元测试：Cursor 项目+文件、会话标题、Edge 页标题

### 2. `timeline/mod.rs`

- `SegmentContext` 扩展：`exe_path`、`aggregation_key`、`enrichment`、`insights`、`recent_app_insights`
- `build_context` 调用 `enrich_segment`，并查询 `recent_for_app`（同应用、片段开始前最多 3 条）
- `chunk_json` 向模型发送：`app_kind`、`parsed_context`、`context_hints`、`hybrid_insights`、`recent_same_app_insights` 等
- 本地 fallback 优先使用 `enrichment.hints`

### 3. `db/insights.rs`

- 新增 `InsightDetail`（含 `window_title`）
- `insights_for_segment` 返回完整字段
- 新增 `recent_for_app(aggregation_key, before_iso, limit)`

### 4. 提示词

- `prompts/timeline-segment-describe.md` 说明各 JSON 字段含义

### 5. JSON 审计

- `TimelineAiRequestLog.segments` 为完整 `SegmentContext`，自动包含 enrichment

## 缓存说明

`cache_key` 仍为 `started_at|ended_at|aggregation_key|window_title`，结构变化不自动失效旧缓存；新片段或标题变化会重新生成。
