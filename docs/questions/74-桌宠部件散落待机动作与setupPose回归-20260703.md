# 桌宠部件散落：待机动作误选 + setToSetupPose 回归

**日期**: 2026-07-03  
**主题**: 柴郡 Spine 部件仍散落

## 现象

用户已在 ViewerEX 配置 `.config.json` 后曾修复散落；选择随机动作 / 立即更新后仍回到部件散落状态。

## 根因（两层）

1. **待机动作误选**  
   柴郡 `chaijun.skel` 无名为 `idle` 的动画，实际待机为 `normal` / `stand`。  
   同步动作列表时按字母排序，旧逻辑 fallback 到首项 **`dance`**，该动画无法从 bind pose 拼合部件 → 视觉上「全部散落」。

2. **`setToSetupPose` 回归（doc 72 方向有误）**  
   此前为「修复散落」加入 `clearTracks + setToSetupPose`，但 chaijun 的 **setup/bind pose 本身就是散开的**；ViewerEX 靠播放 `normal` 等 idle 动画组装。  
   每次切待机若 reset 到 setup pose，会再次散开。

## 修复

### `spinePet.ts`

- 移除 `setToSetupPose` / `resetSkeletonPose`
- 加载后播放 idle 并 `pumpAnimationState` 预推进动画
- 切回待机用 `returnToIdle(mix)`，仅 mix，不 reset skeleton
- 使用 `pickIdleAnimation()` 优先 `normal` → `stand` → 含 idle/stand/normal

### `viewerExApply.ts`

- `pickIdleAnimation` / `isLikelyIdleName` / `applyViewerExBindings` / `pumpAnimationState`

### `models.rs`

- `detect_idle_animation()` 同上优先级
- 若 DB 中待机等于排序首项且不像 idle（如 `dance`），自动修正为 `normal`
- 内置柴郡 `config_file` 改为 `config.json`（非 dotfile，便于静态服务）

### `main.ts`

- 首次 `syncAnimations` 后的 `configureAnimations` 使用 `{ soft: true }`，避免重复打断已组装的待机

## 验证

1. 重启 dev 或设置页点 **立即更新**
2. 柴郡应正常站立，部件不散落
3. 勾选随机动作 → 播放结束后回到 `normal` 待机，仍不散落
