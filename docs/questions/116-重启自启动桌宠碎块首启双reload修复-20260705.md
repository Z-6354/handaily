# 重启/自启动后桌宠碎块：首启双 reload 竞态修复

**日期**：2026-07-05  
**分类**：桌宠 / 启动

## 现象

系统重启或 `--tray` 登录自启动后，桌宠显示为 Spine 碎块；手动「关闭桌宠再打开」可恢复。

## 根因

与 doc 79 同类 **reload 竞态**，发生在首启时序：

1. `sync_on_startup` → `show_pet` 在 `pet.html` 尚未 `Finished` 时就 `show()`
2. 页面加载完成时窗口已可见 → `on_page_load` 发一次 `pet-reload`
3. `schedule_pet_reload_after_show` 40ms 后再发一次 `pet-reload`
4. 两次 `reloadPet`/`initSpine` 重叠 → Spine 在销毁/重建中组装 → 碎块

前端 `bootPetWindow` 2.5s 兜底 reload 在慢机/自启时可能叠加第三次 reload。

## 修复

### Rust `pet/mod.rs`

- `PetRuntimeState` 增加 `page_load_finished` / `page_load_count`
- `on_page_load`：仅 **第二次及以后** 且窗口可见时补发 reload（HMR/重导航）；首启不 emit
- `schedule_pet_reload_after_show`：先 `wait_pet_page_ready`（最长 20s），再延迟 180ms 后 nudge
- `sync_on_startup`：`--tray` 自启 boot 延迟 1200ms，普通启动 400ms

### 前端 `pet/main.ts`

- 移除 2.5s 兜底 `reloadPet()`，避免与 Rust nudge 双重重载

## 验证

1. 完整 `npm run tauri:build` 后安装/运行
2. 开启登录自启动 → 重启或注销再登录 → 桌宠应完整显示
3. 手动启动、切换模型、设置页预览动作 → 无碎块
