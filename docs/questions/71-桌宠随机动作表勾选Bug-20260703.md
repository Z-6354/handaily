# 桌宠随机动作表勾选 Bug 修复

**日期**: 2026-07-03  
**主题**: 随机动作表勾选后状态错乱 / 桌宠异常

## 现象

在设置页「随机动作表」勾选/取消动作后，可能出现：
- 连续勾选多个动作时，只有最后一个生效
- 桌宠闪烁、重载甚至短暂不可见

## 根因

1. **React 闭包过期**：checkbox `onChange` 用 `petRandomAnimations` 直接拼 `next`，快速连点时 state 未刷新，后一次覆盖前一次。
2. **保存后全量重载**：`saveRandomAnimations` 调用 `petNudge` → `pet-reload` → `initSpine` 销毁并重建 Spine，勾选每一项都触发一次，易竞态。

## 修复

### PetPanel

- 用函数式 `setPetRandomAnimations(prev => …)` + `ref` 保存最新列表/频率
- `scheduleRandomSave` 防抖 + `randomSaveSerial` 串行 IPC
- 保存成功后用 API 返回的 `meta` 回写 UI
- 改用 `petRefreshAnimations` 替代 `petNudge`

### 后端 / 桌宠窗

- 新增 `pet_refresh_animations` IPC，emit `pet-animations-changed`
- `main.ts` 监听后仅 `configureAnimations`，不重建 Spine
- `SpinePet`：点击动作播放期间不触发随机动作，避免动画 listener 冲突

## 验证

1. 快速连续勾选多个随机动作 — 全部保留且保存正确
2. 桌宠不闪烁、不重载 Spine
3. 重启后随机动作配置与勾选一致
