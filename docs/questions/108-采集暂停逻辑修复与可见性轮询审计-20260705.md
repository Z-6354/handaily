# 108 · 采集暂停逻辑修复与可见性轮询审计

**日期**：2026-07-05  
**标签**：逻辑完整性、暂停采集、segment、性能

## 用户关切

应用逻辑是否完成 + 性能优化，两者都重要。

## 审计发现

### 逻辑缺陷（已修复）

**暂停采集未闭合进行中的 segment**

- 现象：`set_tracking_enabled(false)` 只关 DB session，poller 停采样但 `open_segment` 仍留在内存。
- 后果：恢复采集后同一段从暂停前 `started_at` 继续延长，**暂停时长被算进工作/音频时间**。
- 修复：`writer::pause_tracking()` — 落盘 open + pending、清 `last_snapshot`、flush 键鼠指标；在 `set_tracking_enabled` 关闭时调用。
- 音频：`audio_monitor` 检测 tracking 由 true→false 时 `flush_all_audio`，暂停后不再延续后台音频段。

### 前端轮询（已优化）

- 生产包主窗口默认 `visible: false`，启动时不应无意义 IPC。
- 原实现每 5s 调用 `isVisible()` IPC，成本高且与托盘 hide 不同步。
- 修复：Rust 在 hide/show 时 emit `main-window-visible`；前端用 ref 门控轮询，**零额外 IPC**；首次 mount 仅一次 `isVisible()` 对齐状态。
