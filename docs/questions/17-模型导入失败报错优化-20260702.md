# 17 · 模型导入失败报错优化

**日期**：2026-07-02  
**类型**：UX 优化

## 需求

模型导入或供应商测试失败时，不要直接展示原始 HTTP/JSON 字符串，改为更优雅、可操作的提示。

## 实现

### `src/lib/apiErrorMessage.ts`

- `parseApiError(raw, context)`：解析 Tauri 抛出的原始错误
- 识别常见场景：未配置密钥、401/403/404、连接失败、空列表
- 火山 Agent Plan / Coding Plan / 按量计费分别给出对应 hint
- `parseTestSuccess`：拆分测试成功消息中的保存统计

### `src/components/SettingsFeedbackBanner.tsx`

结构化反馈组件：图标 + 标题 + 详情 + 操作建议（hint）

### `SettingsPanel.tsx`

- `importFeedback` 按 text/vision 分列显示在各自「导入模型」按钮下方
- `vendorFeedback` 替换原 `vendorTestMsg` 字符串

### 样式

`.settings-feedback` 系列：loading / success / error 配色与紧凑模式
