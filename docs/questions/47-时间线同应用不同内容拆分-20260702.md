# 47 · 时间线同应用不同内容拆分

**日期**：2026-07-02  
**类型**：时间线 / 追踪  
**问答条数**：1

## 问答一

### 问题

优化时间线：同一应用下也可以有不同的内容（例如在 Cursor 先写 A 项目、再写 B 项目，应显示两条而不是一条）。每次向 AI 发送「当前在做什么」和「之前在同一应用里做过什么」；内容相同则不重复添加，不同则在历史里追加。

### 答复

#### 活动内容标识 `activity_key`

新增 `hanpet/src-tauri/src/tracker/activity_key.rs`，从窗口标题/结构化解析提取「在做什么」：

| 应用类型 | 标识来源（优先级） |
|----------|-------------------|
| Cursor / IDE | 标题中的项目名 `project_or_context`，忽略 exe 误推的 `cursor.exe` |
| 浏览器 | 网页标题 / 站点 |
| 聊天 | 聊天对象/群名 |
| 其他 | 标题主段 / 原始标题 |

同一项目内换文件（如 `main.rs` ↔ `utils.rs` 同属 HANDAILY）→ **同一** `activity_key`，仍合并为一条。  
换项目（HANDAILY → OTHER）→ **不同** key，时间线拆成两条。

#### 采集切分

`SnapshotKey` 改为 `aggregation_key + activity_key + is_idle`（不再用原始 `window_title` 字面量）。  
短片段合并（<2s）也要求 **同一 activity_key**，避免快速切项目时误合并。

#### 时间线查询合并

`merge_timeline_segments` 按 `app + activity_key` 合并相邻段（checkpoint 产生的同内容多行仍会合并）。  
返回条目附带 `activity_label`（如项目名）供前端展示。

#### AI 简介上下文

`plan_describe` 按时间顺序为每个时段附带 `prior_activities_in_app`：同一应用内、早于当前、且 **activity_key 不同** 的活动列表；相同内容不重复入历史。  
提示词 `timeline-segment-describe.md` 已补充字段说明。

#### 前端

时间线卡片显示 `activity_label` 标签；本地 fallback 简介优先用活动内容而非整段窗口标题。
