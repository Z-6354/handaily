# 102 · 输入迟滞类 Bug 整体审计

**日期**：2026-07-05  
**标签**：审计、性能、Hook、Raw Input、后台线程

## 审计范围

排查与「启动后鼠标/输入迟滞」同类问题：全局 Hook、高频输入拦截、后台线程抢占、采集关闭后仍重负载。

## 结论摘要

| 类别 | 状态 | 说明 |
|------|------|------|
| 低级别 Hook（WH_*_LL） | ✅ 已清除 | 全库无 `SetWindowsHookEx` |
| 鼠标 Raw Input 移动洪泛 | ✅ 已修复 | 仅键盘走 Raw Input；鼠标改 `GetAsyncKeyState` 边沿轮询 |
| 键盘输入链 | ✅ 安全 | Raw Input 异步 `WM_INPUT`，不在 Hook 链 |
| 后台线程优先级 | ✅ 已统一 | poller/input/audio/file/analysis/period 均 `BELOW_NORMAL` |
| 采集关闭后空转 | ✅ 已优化 | poller/file_watcher 关闭采集时跳过 Win32/notify 处理 |
| 60s segment 拆分 | ✅ 已修复（doc 97） | checkpoint 不再切 open segment |
| 音频 WASAPI 轮询 | ⚪ 低风险 | 2s 间隔 + 采集关时跳过，可接受 |
| 桌宠 Pixi/WebView | ⚪ 独立 | 仅影响桌宠窗口，不拦截系统输入 |
| 时间线/时段 AI 后台 | ⚪ 可接受 | 12s/90s、30s 调度，已降优先级；高峰可能占 CPU 但不阻塞输入 |
| 文件递归监视 | ⚪ 可接受 | Desktop/Documents/Downloads；采集关时休眠 |

## 本轮补充修复

1. **input_monitor**：Raw Input 去掉鼠标注册，避免高 DPI 下每帧 `WM_INPUT`。
2. **poller**：`tracking_enabled == false` 时仅 sleep，不做 Win32 采样与 checkpoint。
3. **analysis / period_scheduler**：worker 与 ticker 线程补充 `dampen_thread_priority`。

## 建议自测

1. 启动后快速移动鼠标、连续打字，应无明显迟滞。
2. 托盘关闭采集后，任务管理器中进程 CPU 应明显降低。
3. 重新开启采集，键鼠统计与前台记录仍正常。
