# 110 · 未绑定 API 密钥时基础功能可用性

**日期**：2026-07-05  
**标签**：API 密钥、基础功能、本地回退、审计

## 结论

**可以。** 未绑定密钥（或未选模型、密码本未解锁）时，**采集、统计、时间线、报告**等基础能力仍可用，只是 AI 增强项降级为本地规则/模板，不会阻断主流程。

例外：**Ollama 本地**无需密码本密钥；**桌宠 AI 台词、Wiki 人设导入、人设测试**等明确依赖 AI 的功能会提示配置或失败。

## 基础功能矩阵（无密钥）

| 功能 | 无密钥时 | 说明 |
|------|----------|------|
| 前台/音频采集、暂停/恢复 | ✅ | 与 AI 无关 |
| 今日概览、热力图、应用记录 | ✅ | 读 SQLite 聚合 |
| 工作时间线列表 | ✅ | segment 落库 |
| 时间线简介 | ✅ 本地 | cache + `timelineDescribe` 走本地 fallback；提示如何绑密钥 |
| 时段小结（今日卡片/热力图） | ✅ 规则 | `rule_based_period`，非 AI 文案 |
| 生成报告 | ✅ 本地 | `compose` 本地 Markdown；`used_ai=false` |
| 历史报告列表/查看 | ✅ | |
| 混合分析（文本） | ✅ | `text::analyze` 本地 |
| 混合分析（截图 vision） | ⚠️ 降级 | 无视觉密钥时用 text fallback |
| 密码本 / 设置 / 桌宠显示 | ✅ | 桌宠动画不依赖 AI |
| 人设测试 / Wiki AI 导入 | ❌ | 需 AI + 密钥（或 Ollama） |
| 桌宠 AI 台词建议/导入 | ❌ | 需 AI；本地格式导入仍可用 |

## 本次修复的 Bug

**已选文本模型但未绑密钥 / 密码本未解锁时，「生成报告」整页报错**

- 原因：`report::prepare_ai_chat` 将 `PreparedTextChat::prepare` 的 `Err`（如「请配置 API 密钥」「密码本未解锁」）直接向上抛，`report_generate` 用 `?` 中断，**未走本地 `compose`**。
- 修复：`prepare_ai_chat` / `prepare_period_chat` 在 prep 失败时 `Ok(None)` + 日志，与 `timeline/describe.rs` 一致，继续本地回退。

## 密钥相关三种状态

1. **未选模型** — 全本地，UI 提示去「设置 → AI 配置」
2. **已选模型、未关联 vault 条目** — 基础功能本地可用；提示关联密钥
3. **已关联但密码本未解锁** — 同上；提示先解锁密码本
