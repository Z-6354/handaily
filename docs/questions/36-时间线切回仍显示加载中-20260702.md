# 36 · 时间线切回仍显示加载中

**日期**：2026-07-02  
**类型**：Bug 修复

## 现象

简介已生成完成，切到其他页再点进「工作时间线」，仍显示「思考中…」或顶部「待生成」提示。

## 根因

1. **组件每次切页卸载**：`{page === "timeline" && <TimelineView />}` 导致离开即销毁，回来重新 `load(0)` 并重设 `pendingDescribe`
2. **pending 与 descriptions 不同步**：`isPending` 仅看 `pendingDescribe`，已有简介但未从 pending 移除时仍显示加载
3. **过期请求**：旧 `timelineDescribe` 完成时 `gen !== loadGenRef` 直接 return，未合并结果也未清理 pending

## 修复

- `App.tsx`：时间线用 `display:none` 保持挂载，切回不重新加载
- `isPending`：仅当 `pendingDescribe` 有且 `descriptions` 尚无该条时才算待生成
- `applyDescribeResult`：无论是否当前 gen，都合并简介；仅当前 gen 完成时清空全部 pending
- 切回时间线时根据 `descriptions` 自动修剪 pending

## 验证

打开时间线等简介完成 → 切到「今日工作」→ 再点「工作时间线」：应直接显示简介，无「思考中…」。
