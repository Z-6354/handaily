# 49 · 时间线简介机器格式与 Skill 口吻修复

**日期**：2026-07-02  
**类型**：时间线 / AI 简介  
**问答条数**：1

## 问答一

### 问题

优化后时间线仍出现机器式简介，如「开发：handaily_桌宠集成_4be8ad40.plan.md - live2d - Cursor · 窗口「…」」，没按 Skill 人设来写。

### 原因

1. **本地 fallback** 直接使用了 `insight_lines`（含 `[text·development·…]` 和重复窗口标题）
2. **text 分析** 的 `build_summary` 输出「开发：{完整 window_title}」
3. **旧 cache** 存了上述机器格式，被当作有效简介复用
4. AI 提示词未明确禁止照抄 window_title

### 修复

| 模块 | 改动 |
|------|------|
| `activity_key.rs` | 新增 `human_activity_summary` / `human_summary_from_enrichment`；`is_machine_summary` 识别旧格式 |
| `analysis/text.rs` | 混合分析摘要用自然语言，去掉「开发：」前缀 |
| `timeline/mod.rs` | fallback 优先 enrichment 人话摘要；过滤旧 cache；简化 insight_lines；`parse_ai_map` 用 json_util |
| `timeline-segment-describe.md` | 强调必须按 Skill 人设、禁止复述 window_title |

### 使用

刷新时间线页，旧机器格式 cache 会自动视为未缓存并重新生成。若仍不对，检查设置里文本模型与人设是否已配置。
