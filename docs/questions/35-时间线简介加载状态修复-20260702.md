# 35 · 时间线简介加载状态仅显示待生成条目

**日期**：2026-07-02  
**类型**：Bug 修复 / UX

## 现象

工作时间线 AI 简介生成时，**所有**卡片都显示「思考中…」/加载样式；实际上只有未缓存的条目在调模型，且要等整批 `timeline_describe` 返回后才会一次性更新。

## 根因

`TimelineView.tsx` 使用全局 `describing`：

- 请求开始即 `setDescribing(true)`，此时 `descriptions` 仍为空
- 每条卡片：`describing && !descByStarted.has(...)` → 全部显示「思考中…」
- 加载样式：`describing && !usedAi` → 本地简介条目也被标为 loading

## 修复

### 后端

- 新增 `timeline_cached`：快速返回当前页已缓存简介
- `timeline_describe` 每处理完一批（最多 20 条）后 `emit("timeline-describe-chunk", { offset, limit, entries })`

### 前端

1. `getTimeline` 后先 `timelineCached` → 已缓存条目立即展示
2. 用 `pendingDescribe: Set<started_at>` 标记**真正待 AI 生成**的条目
3. 监听 `timeline-describe-chunk`，分批合并结果并从未决集合移除
4. 仅 `isPending` 的卡片显示「思考中…」与 loading 样式
5. 顶部提示改为「N 条待生成」

## 验证

重启应用 → 打开活动时间线：有缓存的条目应瞬间显示简介；仅新片段显示「思考中…」，且随批次完成逐条更新。
