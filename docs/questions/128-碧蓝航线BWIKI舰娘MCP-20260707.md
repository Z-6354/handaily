# 128 · 碧蓝航线 BWIKI 舰娘 MCP

**日期**：2026-07-07

## 问题

需要 MCP 从 [舰船图鉴](https://wiki.biligame.com/blhx/%E8%88%B0%E8%88%B9%E5%9B%BE%E9%89%B4) 读取全部舰娘，再逐页抓取台词、情人节礼物、角色设定、立绘/头像/语音等，SQLite 增量缓存，供开发者用 Cursor AI 导入小寒日报人物/性格（**不**接入 HANDAILY 内置 AI）。

## 实现

目录：`mcp/blhx-wiki/`

| 工具 | 作用 |
|------|------|
| `blhx_sync_catalog` | 同步图鉴索引（~865 舰娘） |
| `blhx_sync_ships` | 增量批量抓取（跳过已存在） |
| `blhx_fetch_ship` | 单舰抓取 |
| `blhx_get_ship` / `blhx_search_ships` | 读本地库 |
| `blhx_export_handaily` | 导出 persona 参考文本 + 台词 + 资源 URL |

Cursor 配置：`.cursor/mcp.json`（stdio + `node mcp/blhx-wiki/dist/index.js`）。

> **路径更新（2026-07-12）**：CLI 现通过仓库根 `npm run blhx:import` / `npm run live2d:import` 调用；源码在 `hanpet/src-tauri/`。架构见 [ARCHITECTURE.md](../../ARCHITECTURE.md)。

解析逻辑对齐 `hanpet/src-tauri/src/pet/wiki_scrape.rs`；BWIKI 需带 `Referer` 头，否则可能 HTTP 567。

## 工作流

1. `blhx_sync_catalog`
2. 多次 `blhx_sync_ships` `{ "limit": 20 }` 直至 pending=0
3. `blhx_export_handaily` `{ "name": "欧根亲王" }`
4. 在 HANDAILY 人物页 Wiki/文本导入，或由 AI 写入 `personas/`、`characters/manifest.json`

桌宠 Spine 模型仍须单独导入；MCP 侧重性格与台词资料。
