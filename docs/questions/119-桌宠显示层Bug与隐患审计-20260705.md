# 桌宠显示层 Bug 与隐患审计（118 后续）

**日期**：2026-07-05  
**分类**：桌宠 / 稳定性 / 输入监控

## 问答一：继续检测 bug 与隐患

### 已修复（本回合）

| 问题 | 严重度 | 说明 |
|------|--------|------|
| `resolveIdleName` 忽略 `returnIdleName` | 中 | `ensureIdleLoop` / 启动 idle 只用 `idleName`，与 `playOneShot` 的 `returnIdleName \|\| idleName` 不一致；若两字段不同会导致切回错误待机或 `syncBaseIdleAfterMeta` 反复重设 track |
| Spine 失败仍 `pet_mark_spine_ready` | 中 | `initSpine` catch 吞错后 `reloadPet` 仍置位 → 下次 `show_pet` 走 resume 而 `pet===null`，多一次无效 resume 再 reload |
| Raw Input 未对齐缓冲区 | 高（dev） | `[u8;512]` 强转 `&RAWINPUT` 在 debug 下 misaligned pointer panic（`input_monitor.rs:249`）；改为 `#[repr(C, align(8))]` + 长度校验 |

### 仍存隐患（未改或低优先级）

| 隐患 | 严重度 | 说明 |
|------|--------|------|
| `pet-resume` 与 `visibilitychange` 双触发 | 低 | Rust `schedule_pet_resume_after_show` 与前端 `visibilitychange` 均调 `resumePetFromHidden`，重复 refit，一般无害 |
| `on_page_load` count>1 补发 reload | 低 | WebView 非首载 Finished 且可见时全量 reload；生产环境少见，dev/HMR 可能多 reload 一次 |
| `reloadInProgress` 时 resume 直接 return | 低 | 隐藏期间若 settings 触发 reload，显示瞬间 resume 被跳过；reload 完成后会重建 Spine，通常可接受 |
| `fitSpineToCanvas` 在动作中途 refit | 低 | `resumePetFromHidden` / 窗口 resize 用当前 pose 算 bounds，极端 pose 可能闪一下 |
| `playOneShot` 的 `actionPlaying` 在一动作结束即 false | 低 | mix 回 idle 期间随机调度可能插入，旧有问题，非 118 引入 |
| 用户手动隐藏时 `emit_remark` | 低 | 窗口仍隐藏只排队气泡，需用户再显示桌宠才看到 |
| 静态图 fallback 无 Spine | 信息 | 加载失败显示 png，不标记 `spine_ready`，每次显示会尝试全量 reload（符合预期） |

### 建议验证

1. `npm run tauri:dev` 长时间按键，确认不再 panic  
2. 隐藏 → 显示 → 全屏游戏 → 退出全屏，桌宠 resume 无碎块  
3. 故意损坏 skel 触发 fallback，再隐藏显示，应自动 reload 而非空 resume  
4. 模型 `idle_animation` ≠ `return_idle_animation` 时，动作结束仍回到 return idle
