# 桌宠点击动作应在同步 JSON 时写入

**日期**: 2026-07-03

## 问题

点击动作不应在点击时用正则临时匹配，而应在 `pet_sync_animations` 写入 `model.json` / `pet_anim_meta_*` 时就解析并持久化。

## 修复

- Rust `sync_animations`：`click_animation` 为空时自动检测 `tap/click/touch/hit` 并写入 JSON
- 桌宠加载：`syncAnimations` 返回最新 meta，用返回值 `configureAnimations`（不再用旧 config）
- `playTap()`：只播放已配置的 `clickName`，无点击配置时回退待机，不再运行时正则搜索
