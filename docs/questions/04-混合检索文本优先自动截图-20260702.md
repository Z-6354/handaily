# 04 · 混合检索：文本优先，低置信度自动截图

**日期**：2026-07-02  
**状态**：已实现

## 需求

平时使用文本分析；文本检测无效或效果差时自动切换截图；用户高 CPU 占用时不截图。

## 实现

### 流程

```
segment 落盘 → 分析队列
  → 文本分析（标题 + 应用名 + 规则分类）
  → confidence ≥ 0.45 → 仅存 text insight
  → confidence < 0.45 → 检查守卫 → 截图 → 视觉分析（可选 OpenAI）
  → 截图 JPEG 内存处理，不落盘
```

### 守卫条件（跳过截图）

- 混合分析 / 自动截图关闭
- 应用在排除列表（keepass、bitwarden 等）
- 系统 CPU ≥ 阈值（默认 75%）
- 距上次截图 < 间隔（默认 120s）
- segment 时长 < 5s

### 模块

| 路径 | 职责 |
|------|------|
| `analysis/text.rs` | 文本语义 + 置信度 |
| `analysis/guard.rs` | CPU / 频率 / 排除 |
| `analysis/coordinator.rs` | 异步 worker |
| `screenshot/mod.rs` | 前台窗口 JPEG |
| `analysis/vision.rs` | OpenAI vision（可选） |
| `db/insights.rs` | `activity_insights` 表 |

### 设置（SettingsPanel）

- 混合分析、自动截图、CPU 阈值、截图间隔、视觉 AI
- 今日统计：文本/截图/跳过次数

### IPC

- `analysis_get_status`
- `analysis_list_insights`
