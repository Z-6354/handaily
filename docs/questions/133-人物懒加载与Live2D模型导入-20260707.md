# 133 · 人物列表懒加载与 Live2D 模型批量导入

**日期**：2026-07-07

## 需求

1. 866 人物列表需懒加载/分页，避免一次渲染卡顿
2. `--reference-only` 只导入了文本人设，Live2D 模型未导入

## 实现

### 人物页懒加载

- Rust：`CharacterBrief`、`characters_list_brief`、`characters_list_page`
- 前端：`useCharacterRoster`（每页 48，滚到底自动加载；搜索/收藏时拉全量 brief）
- 人物页顶栏显示「共 866 位 · 已加载 N」

### 模型导入（两步流程）

| 步骤 | 命令 | 内容 |
|------|------|------|
| 1 人设 | `blhx_import --all --reference-only` | 文本 + Skill（无 AI） |
| 2 模型 | `live2d-plan` + `live2d_import` | Spine 三件套 → 皮肤 |

### 2026-07-07 补全与修复

此前「已实现但看不到」的根因：

1. **plan.json 字段名**：MCP 导出为 camelCase（`skinName`），Rust 原期望 snake_case → 导入直接 JSON 解析失败，模型一直只有 5 个内置。
2. **批量导入性能**：每条都 `list_models` + 读写 866 人 manifest → 第二批起极慢像卡死。已改为一次缓存模型名、内存改 manifest、批次末写盘一次。
3. **前端小 bug**：`useCharacterRoster.loadMore` 依赖数组自引用；反馈 `tone: "warning"` 非合法类型。
4. **dev 与 CLI 争用 cargo lock**：需等 `start-dev` 编译完成后再跑 `live2d_import`，或直接在人物页点「导入 Live2D 模型」（每批 50）。

修复后 CLI 全量导入约 1444 模型（`pet-models` 约 1486 含历史/内置）：

```powershell
Copy-Item D:\0HAN\HANDAILY\mcp\blhx-wiki\plan.json `
  $env:APPDATA\xiaohan-daily\data\live2d-plan.json

cd D:\0HAN\HANDAILY\src-tauri
cargo build --bin live2d_import
.\target\debug\live2d_import.exe --plan ..\mcp\blhx-wiki\plan.json --all
# 或 UI：人物页 → 「导入 Live2D 模型」重复点击直至剩余 0
```

重启应用后人物页应显示 **共 866 位**，滚动加载；详情里可见多皮肤。
