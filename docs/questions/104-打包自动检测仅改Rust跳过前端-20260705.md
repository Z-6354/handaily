# 104 · 打包自动检测仅改 Rust 跳过前端

**日期**：2026-07-05  
**标签**：构建、自动检测、tauri-before-build

## 需求

不必手动设 `SKIP_FE_BUILD=1`；打包时自动判断是否可以跳过 vite/tsc。

## 逻辑

`scripts/tauri-before-build.mjs` 在 `tauri build` 前：

1. 扫描前端输入最大 mtime：`src/`、`index.html`、`pet.html`、`vite.config.ts`、`tsconfig.json`、`package.json`、`public/` 等。
2. 检查 `dist/index.html` 与 `dist/pet.html` 是否存在。
3. 若 dist 最旧文件 mtime **≥** 全部前端输入最大 mtime → 跳过构建，打印「仅 Rust/后端改动」。
4. 否则执行 `vite build`（或 `FE_BUILD_FULL=1` 时 `tsc + vite`）。

## 手动覆盖

| 变量 | 作用 |
|------|------|
| `SKIP_FE_BUILD=1` | 强制跳过（dist 须存在） |
| `FORCE_FE_BUILD=1` | 强制重建前端 |
| `FE_BUILD_FULL=1` | 全量 tsc + vite，且检测范围含 tsconfig.app/node |

## 说明

- 只改 `src-tauri/`、`personas/`、`prompts/` 等不在监视列表内的路径时，会自动跳过前端。
- 改了 `package-lock.json` 会触发重建（依赖变更）。
