# 105 · 项目运行时与轮询继续优化

**日期**：2026-07-05  
**标签**：性能、IPC、DB、后台线程

## 改动摘要

### 前端（`App.tsx`）
- 5s 全量 IPC 拆为 **轻量**（overview + status + workTypes）与 **重量**（heatmap + breakdown + periods）。
- 重量数据：相关页面每 30s 刷新；切到今日/热力图/应用记录时立即拉一次。
- 时间线、设置等页不再每 5s 扫 3 日热力图。

### 后端
| 项 | 文件 | 说明 |
|----|------|------|
| 空闲阈值缓存 | `state.rs`, `poller.rs`, `commands.rs` | 去掉 poller 每 2s 读 DB |
| 应用图标批量查询 | `db/stats.rs`, `commands.rs` | `latest_exe_paths_for_keys` 替代 N+1 |
| 采集暂停 | `input_monitor.rs`, `file_watcher.rs` | 深睡眠 + 清空 notify 积压 |
| AI 后台 | `writer.rs`, `timeline/scheduler.rs`, `period_scheduler.rs` | 暂停采集时不 enqueue / 不跑时段与时间线 AI |
| 音频前台 PID | `audio_monitor.rs`, `tracker/mod.rs` | 复用 poller 缓存的 `foreground.pid` |
| 构建脚本 | `scripts/build.ps1` | 修正 `release-fast` 产物路径提示 |

## 预期效果

- 主窗口在「时间线 / 设置」等页：IPC + DB 负载约降 **60%+**。
- 暂停采集：输入线程、文件监视、AI 调度显著减活。
- poller：减少 **~30 次/分钟** 无谓 DB 锁竞争。
