# 103 · 导出 exe 构建加速

**日期**：2026-07-05  
**标签**：构建、Tauri、release-fast

## 需求

缩短 `npm run tauri:build` 导出 exe / 安装包的总耗时。

## 瓶颈

1. Rust `release` 开启 `thin LTO` + 低 `codegen-units`，链接阶段耗时长（约数分钟）。
2. 每次打包都跑 `tsc --noEmit && vite build`，前端未改时也重复。
3. NSIS 打包装在完整 Rust 编译之后，无法省略时至少应加速 Rust 段。

## 改动

| 项 | 说明 |
|----|------|
| `[profile.release-fast]` | `lto=false`、`codegen-units=256`，默认 `tauri:build` 使用 |
| `release` + `tauri:build:full` | 保留 LTO，正式发布用 |
| `scripts/tauri-before-build.mjs` | 默认仅 `vite build`；**自动对比** `src/`、`index.html` 等与 `dist/` 时间戳，仅改 Rust 时跳过 |
| `vite` | `reportCompressedSize: false` 减少构建尾部耗时 |
| `.cargo/config.toml` | 开启 incremental |
| `scripts/build.ps1` | `-ExeOnly`、`-SkipFe`（强制跳过）、`-Full` |

## 自动跳过前端

打包前脚本会比较前端输入（`src/`、`vite.config.ts`、`index.html` 等）与 `dist/index.html`、`dist/pet.html` 的修改时间。若 **dist 齐全且比所有前端输入都新**，则判定为「仅 Rust/后端改动」，跳过 vite。

手动覆盖：`SKIP_FE_BUILD=1` 强制跳过；`FORCE_FE_BUILD=1` 强制重建。

## 命令速查

```bash
npm run tauri:build          # fast + NSIS（自动跳过未改动的 dist）
npm run tauri:build:exe      # 仅 exe，最快
npm run tauri:build:full     # LTO + tsc 正式版
```

## 产物路径

- exe: `src-tauri/target/release/xiaohan-daily.exe`
- NSIS: `src-tauri/target/release/bundle/nsis/`
