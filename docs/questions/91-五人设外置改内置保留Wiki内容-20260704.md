# 五人设外置改内置，保留 Wiki 导入原始内容

**日期**：2026-07-04  
**分类**：persona / 内置资源

## 问题

用户要求将当前 5 个 AI 人设（柴郡、恶毒、五十铃、企业、塔什干）像桌宠模型一样**内置进 exe**，但明确强调：

- **Skill `.md` 与结构化 `.json` 内容不变**（使用 AppData 中 Wiki 导入的原始文件）
- 仅将「外置文件」改为「`include_str!` 内置 + 启动时种子到用户目录」
- 不要重写或简化人设文案

## 数据源

用户 AppData：`%APPDATA%\xiaohan-daily\data\personas\`

| 内置 slug | 名称 | 来源文件（哈希 ID） |
|-----------|------|---------------------|
| `cheshire` | 柴郡 | `cheshire.*` |
| `edu` | 恶毒 | `p951a05aa.*` |
| `wushiling` | 五十铃 | `pe2795090.*` |
| `qiye` | 企业 | `pc5623cfa.*` |
| `tashigan` | 塔什干 | `pea9d211a.*` |

## 实现

1. **复制原始文件到仓库 `personas/`**  
   用 PowerShell 从 AppData 按上表复制 `.md` / `.json` 到 slug 文件名（如 `p951a05aa.md` → `edu.md`），不改动内容。

2. **`manifest.json`**  
   五人设条目；`description` 取自各 `.json` 的 `introduction` 字段（与 Wiki 导入一致）。

3. **`hanpet/src-tauri/src/persona/mod.rs`**  
   - `EMBEDDED_PERSONAS` / `EMBEDDED_PROFILES`：`include_str!` 引用上述 5 组文件  
   - `LEGACY_BUILTIN_PERSONA_IDS`：同时支持人设 `p` 前缀哈希与桌宠 `m` 前缀哈希 → slug  
   - `migrate_legacy_personas`：manifest 合并、active ID 修正；`sync_embedded_builtin_files` 将内置内容写入用户目录；`remove_legacy_persona_files` 删除旧哈希文件

## 验证

```bash
cd hanpet/src-tauri && cargo test persona::tests
```

重建 Tauri 后，新安装或启动迁移会将 AppData 中的哈希 ID 人设合并为 slug，并用仓库内 Wiki 原始 Skill/JSON 覆盖内置五人设文件。
