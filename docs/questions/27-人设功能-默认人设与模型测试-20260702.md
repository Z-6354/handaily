# 27 · 人设功能：默认人设与模型测试

**日期**：2026-07-02  
**类型**：新功能

## 需求

- 提供若干默认人设（碧蓝航线柴郡、鸣潮菲比、缘之空穹等），以 Markdown 存放
- 系统语气与总结风格随所选人设变化
- 设置页可测试「人设 + 当前文本模型」是否连通

## 实现

### 数据与种子

| 路径 | 说明 |
|------|------|
| `personas/manifest.json` | 人设元数据（id、名称、出处、简介） |
| `personas/*.md` | 人设全文，作为 LLM **system prompt** |
| `%AppData%/xiaohan-daily/personas/` | 首次启动种子复制；用户可编辑 |

默认人设：`default`（小寒助手）、`cheshire`、`phoebe`、`sora`。激活 ID 存 SQLite `active_persona_id`。

### 后端

- `src-tauri/src/persona/mod.rs`：加载 manifest、读 md、`system_prompt()`
- `ai/adapter.rs`：`chat_text` / `chat_vision` 自动注入当前人设
- IPC：`persona_list`、`persona_set_active`、`app_get_personas_path`、`ai_test_persona`
- 测试用 user prompt：`prompts/persona-test.md`（要求 2～3 句中文自我介绍，非 JSON）

### 前端

- 设置 → AI 配置：人设卡片网格、切换激活、「测试人设与模型」按钮
- `xiaohan.ts`：`PersonaInfo`、`PersonaTestResult` 及对应 invoke

### 使用

1. 配置供应商密钥并选择文本模型  
2. 设置页选择人设  
3. 点击「测试人设与模型」——成功时展示模型按人设口吻的回复  

时段分析、截图理解等 AI 调用均会带上当前人设 system prompt；任务 prompt（如 `period-analysis.md`）仍为 user 消息。
