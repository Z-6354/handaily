# 132 · 人物列表未随 persona 导入更新

**日期**：2026-07-07

## 现象

批量导入后人物页仍只显示 5～11 个角色，看不到几百个新角色。

## 根因

1. **Live2D 计划 ≠ 人物**：`live2d-plan` 的 1400+ 条是模型匹配计划，须先 `blhx_import` 导入人设才会出现在人物页。
2. **数据已导入但 UI 未刷新**：CLI 导入后应用未重启/未重新拉列表，仍显示旧状态。
3. **列表加载性能**：866 人物时 `list_characters` 对每人重复扫描模型目录，首次加载极慢，像「只有几个」。

本地实测：`hanpet/bundled/roster/personas/manifest.json` 与 `characters/manifest.json` 均为 **866** 条。

## 修复

- `sync_character_manifest_from_personas`：persona → character manifest 同步
- `blhx_import --all --reference-only`：无 AI 快速批量导入（约 855 舰娘/次）
- `blhx_import --sync-characters`：仅同步 manifest
- `list_characters`：模型列表只读一次（缓存）
- 人物页：显示「共 N 位」、**刷新列表**按钮、窗口聚焦/可见时自动刷新

## 使用

```powershell
cd D:\0HAN\HANDAILY\src-tauri
cargo run --bin blhx_import -- --sync-characters
```

然后**重启小寒日报**（或打开人物页点「刷新列表」）。左上角应显示 **共 866 位**，网格内**向下滚动**浏览全部角色。

若仍只有几个：检查是否开启「收藏」筛选；清除搜索框。
