# 项目隐患与优化审计

## 问题

用户要求检测其他项目 Bug 或隐患，并给出优化建议。

## 审计范围

`src-tauri/src`（Rust 核心）、`src/`（React 主应用）、`src/pet/`（桌宠）。

## 高优先级（建议尽快修）

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| H1 | **截图分析持锁调 AI** | `analysis/coordinator.rs` L121–127 | `db.lock()` 贯穿 `vision::analyze_screenshot()` 网络 I/O，阻塞采集落库、IPC、退出 flush |
| H2 | **聚合器与 DB 不一致** | `tracker/writer.rs` L125–146 | `apply_to_aggregator` 先于 `insert_segment`，插入失败被 `let _` 吞掉，今日统计永久偏差 |
| H3 | **午夜整点时段日期错误** | `analysis/period_scheduler.rs` L110–131 | `00:00` 时 `prev_hour=23` 但 `date_key` 用今天，会分析「今天 23:00」而非「昨天 23:00」 |
| H4 | **时间线后台自动 AI** | `App.tsx` L187–189 + `TimelineView.tsx` L240–244 | 组件始终挂载，`load(0)` 启动即 `timelineDescribe`，未打开时间线也跑 AI、写缓存、弹桌宠气泡 |
| H5 | **桌宠 reload 并发** | `pet/main.ts` + `PetPanel.tsx` L439–443 | 滑块每次 `onChange` 触发 `petReload()`，可并行 `initSpine()`，WebGL/Spine 状态易乱 |
| H6 | **全局 refresh 竞态** | `App.tsx` L96–124 | 5s 轮询无 generation 令牌，慢请求可覆盖新数据 |

## 中优先级

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| M1 | DB 锁毒化恢复不一致 | `db/mod.rs` vs `poller/writer/commands` | 部分路径 `lock_conn()` 恢复，热路径 `unwrap()`，后台 panic 后 IPC 行为分裂 |
| M2 | `timeline_describe` 无并发锁 | `ipc/commands.rs` | 并行调用重复 AI 批次、浪费额度 |
| M3 | 分析队列满静默丢任务 | `analysis/coordinator.rs` L46–51 | `try_send` Full 无日志，洞察缺失 |
| M4 | 崩溃恢复段时长归零 | `state.rs` L111–115 | 未闭合 segment 直接 `duration_ms=0`，最多丢一个 checkpoint 窗口 |
| M5 | 每次同步 AI 新建 Tokio runtime | `ai/adapter.rs` | 时段/分析 worker 重复建 runtime，额外开销 |
| M6 | 时间过滤器仅筛当前页 | `TimelineView.tsx` L269–285 | 分页后 30m/1h/2h 过滤结果不完整 |
| M7 | 无用 metrics 轮询 | `App.tsx` L104 | `getTodayMetrics()` 每 5s 拉取但 state 丢弃 |
| M8 | 桌宠设置乐观更新无回滚 | `PetPanel.tsx` | 省电/缩放失败不回滚本地 state |
| M9 | 输入钩子失败无提示 | `tracker/input_monitor.rs` | `SetWindowsHookExW` 失败静默，键鼠统计恒为 0 |

## 低优先级

| # | 问题 | 位置 |
|---|------|------|
| L1 | 桌宠窗口创建无互斥 | `pet/mod.rs` `ensure_pet_window` |
| L2 | 调度器 `scheduler_running` 永不复位 | `pet/mod.rs` |
| L3 | 时间线编辑/删除按钮常显 disabled | `TimelineView.tsx` |
| L4 | pet 事件 listen 无 unlisten（HMR 重复） | `pet/main.ts` |
| L5 | 截图 AI 失败伪装成本地低置信洞察 | `analysis/vision.rs` |

## 建议修复顺序

1. **H4** — `TimelineView` 仅在 `active` 时 `load`，或改为条件挂载  
2. **H1 + H2** — 释放 DB 锁后再调 AI；`insert_segment` 失败不更新聚合器  
3. **H3** — 午夜 `prev_hour` 时 `date_key` 回退到昨天  
4. **H5 + H6** — 滑块 `onMouseUp`/debounce 保存；`refresh` 加 `loadGenRef`  
5. **M2** — `timeline_describe` 加 `Mutex` 或 in-flight 标志  

## 答复摘要

本轮为只读审计，未改代码。上述 6 项高优问题对数据准确性、AI 成本与用户体验影响最大，建议按序分批修复。
