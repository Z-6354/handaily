# 异常检测批量修复

## 问题

桌宠/时间线/主窗口 ACL 修复后，继续审计并修复其余异常行为。

## 已修复项

### 1. 时间线 describe 锁饥饿

**现象**：后台调度器持有 `timeline_describe_lock` 整轮 AI 生成，前端打开时间线时 `timeline_describe` 长时间阻塞。

**修复**：锁改为在 `describe_chunks` 内按每批（≤20 段）加锁，AI 请求在锁外执行；调度器与 IPC 入口不再整轮占锁。

### 2. 后台 AI 桌宠气泡刷屏

**现象**：后台 `run_today_uncached` 每批 AI 完成都 `emit_remark`，桌宠频繁弹气泡。

**修复**：`describe_chunks` 增加 `emit_pet_remarks`；仅 `run_page_describe`（用户主动打开时间线）为 `true`，后台为 `false`。

### 3. 桌宠设置保存过重 reload

**现象**：改缩放/省电/备注间隔/动作时调用 `pet_reload`，Rust 端销毁并重建 WebView，易闪烁且与 `tauri.conf` 预创建窗口冲突。

**修复**：
- 新增 `pet_nudge` IPC + `nudge_pet()`，仅向桌宠窗口 emit `pet-reload`
- `reload_pet` 改为窗口存在时只 nudge，不 `destroy_pet_window`
- `PetPanel` 轻量设置改用 `petNudge()`；换模型等仍走 `petReload()`

### 4. 桌宠禁用时预创建窗口仍可见

**现象**：`tauri.conf.json` 预注册 pet 窗口，`pet_enabled=0` 时 startup 直接 return，窗口可能残留。

**修复**：`sync_on_startup` 在 disabled 时显式 `win.hide()`。

### 5. 时间线组件常驻挂载

**现象**：`TimelineView` 用 CSS `display:none` 隐藏但仍挂载，离开页面后仍可能持有 listener/定时逻辑。

**修复**：`App.tsx` 改为 `{page === "timeline" && <TimelineView active />}` 条件挂载。
