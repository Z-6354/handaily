# 82 — 桌宠切换模型加载加速

**日期**: 2026-07-04  
**主题**: 切换模型等待时间过长

## 现象

桌宠页下拉切换模型需数秒，体验明显慢于预期。

## 根因

1. **主因**：`set_active_model` 走 `restart_pet_window`（销毁 WebView → 等待关闭 → 重建 → 重载 600KB+ 桌宠脚本 + Pixi 初始化），成本远高于换 Spine 资源。
2. **次因**：导入模型经 `pet_read_model_asset` 逐文件 IPC + base64；`syncAnimations` 阻塞首帧；切换路径仍跑完整 `finalizePetAssembly`。

## 修复

| 层级 | 改动 |
|------|------|
| **Rust** | 切换模型改为 `nudge_pet` 同窗口热重载；新增 `pet_read_model_bundle` 批量读资源 |
| **前端** | 全局 Blob 缓存 + `preloadModelAssets`；切换快路径跳过阻塞 sync、缩短组装 pump |
| **Spine** | `skipBootAnimation` 时缩短 assemble/warm pump 步数 |

切换后再切回已加载模型时，资源命中内存缓存，更快。

## 验证

重新编译运行；桌宠可见时在模型间多次切换，应在 1 秒内完成（视 PNG 大小而定）。若出现碎块，报告切换方向。
