# 106 · 启动与前端分包继续优化

**日期**：2026-07-05  
**标签**：启动、懒加载、代码分割、后台任务

## 改动

### 启动加速（`lib.rs` / `state.rs`）
- 时间线 AI 历史日志 `consolidate_past_days_on_startup` 从 `AppState::new` 同步路径移到 **`startup-maint` 后台线程**，不阻塞窗口首显。

### 前端（`App.tsx` / `vite.config.ts`）
- **React.lazy** 懒加载：报告、历史、Agent、人设、桌宠、密码本、设置、性能检测等次要页面。
- **manualChunks**：`vendor-react`、`vendor-tauri` 独立 chunk，主包更小、首屏解析更快。

### 后台 AI（`period_scheduler.rs` / `coordinator.rs`）
- 时段调度 `check_triggers` 在采集暂停时不排队任务。
- 混合分析 `process_job` 采集暂停时直接返回（队列中遗留任务也不执行）。

## 预期

- 冷启动到主窗口可交互更快（尤其 timeline-ai 日志多的用户）。
- 主窗口 JS 首包体积下降，常用页（今日/时间线/热力图）加载更轻。
- 暂停采集后 AI  worker 完全空闲。
