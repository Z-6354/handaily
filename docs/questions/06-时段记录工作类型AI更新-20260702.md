# 06 — 时段记录样式优化 + AI 工作类型定时更新

**日期**：2026-07-02  
**标签**：热力图、工作类型、AI 时段分析、调度

## 需求

1. 时段记录变窄变长：每天 24 小时一条细横线，三天叠放（非大块三行）
2. 工作类型：开发、会议、沟通、文档、娱乐、游戏、休闲等，用户可自定义
3. AI 定时更新：启动 5 分钟、每小时整点、长时间切换应用后，将时段活动发给 AI 判断类型并总结

## 实现

### 前端

- `HeatmapStrip.tsx`：细条热力图（12px 高 × 24 列），工作类型着色 + 强度回退
- `TodayDashboard`：时段记录独立横条；新增「AI 时段总结」列表
- `SettingsPanel`：工作类型管理（改色、添加/删除自定义）

### 后端

- `work_type/`：类型配置 JSON 存 `app_settings.work_types_config`
- `db/periods.rs`：`period_summaries`、`hour_work_types` 表
- `analysis/period.rs`：文本模型分析时段 segments
- `analysis/period_scheduler.rs`：5 分钟 / 整点 / 长会话（≥15min）触发
- `stats.rs`：热力图返回 `work_types`、`summaries` 每 hour

### IPC

- `work_types_get` / `work_types_save`
- `period_list_summaries`

## 触发说明

| 触发 | 条件 |
|------|------|
| startup_5m | 进程启动满 5 分钟 |
| hourly | 每小时 0–2 分分析上一小时 |
| long_session | 单应用连续使用 ≥15 分钟后切换 |

需配置文本 AI 模型（设置 → AI 模型）；无 API 时回退规则推断。
