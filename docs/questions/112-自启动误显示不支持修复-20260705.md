# 自启动误显示「不支持开机自启动」修复

**日期**：2026-07-05  
**分类**：设置 / 启动

## 现象

设置 → 通用 → 启动 中，Windows 桌面版有时显示「当前系统暂不支持开机自启动」，开关被禁用。

## 根因

`autostartGetStatus` 调用失败时，`catch` 分支把 `autostartSupported` 设为 `false`，与真实平台能力无关。

在浏览器打开 Vite 预览（`localhost:1420`）时 invoke 不可用，也会触发该分支。

## 修复

- 新增 `src/lib/platform.ts`：用 Tauri 运行时 + Windows UA 判断自启动是否可用
- `SettingsPanel`：初始值与 API 失败回退均用客户端平台检测，不再一律 `false`
- 非 Tauri 环境提示「请在桌面客户端中配置登录自启动」
- Rust `autostart_get_status` 的 `supported` 改为 `platform_supported()` 运行时判断
- **注册表写入改为 Win32 API**（替代 `reg.exe`），修复路径含空格或 `--tray` 参数时写入失败；注册表失败时回滚 DB
