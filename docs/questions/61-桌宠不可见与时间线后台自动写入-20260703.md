# 桌宠不可见与时间线后台自动写入

## 问题

1. **桌宠依旧看不到**：启动后桌宠窗口不出现或透明无内容。
2. **时间线需点击查看才开始加载**：`TimelineView` 仅在 `active={page === "timeline"}` 时调用 `timelineDescribe`，未打开时间线页不会生成/写入 AI 简介缓存。

## 根因

### 桌宠

1. **全屏误判（最大化）**：`is_foreground_fullscreen()` 仅排除 `SW_SHOWMAXIMIZED`，无边框最大化（`showCmd=normal` 但矩形贴满显示器）仍被判为全屏 → `sync_fullscreen_visibility` 立刻 `hide_pet`。
2. **开发模式竞态**：桌宠 URL 为 `http://localhost:1420/pet.html`，`sync_on_startup` 在 Vite 就绪前创建窗口 → 加载失败、透明空窗。
3. **本进程前台**：小寒日报主窗口最大化前台时，不应触发全屏隐藏逻辑。

### 时间线

- 审计项 H4 曾将「仅 active 时 load」作为**省 AI 成本**的修复方向；用户实际需求是**后台自动写入**，与「打开页才 describe」相反。
- 正确做法：Rust 后台调度器在进程启动后周期性 `run_today_uncached`，与 UI 是否打开时间线无关。

## 修复

### 桌宠（Rust）

- `win32::is_foreground_fullscreen`：增加 `IsZoomed(hwnd)` 判断，最大化窗口一律不算沉浸式全屏。
- `win32::is_foreground_own_process`：前台 PID == 本进程时不触发全屏隐藏。
- `pet::sync_on_startup`：debug 模式下异步重试最多 20 次（每秒）`show_pet` + `pet-reload`，等待 dev server。

### 时间线（Rust）

- 抽出 `timeline/describe.rs`：`run_page_describe`（IPC）、`run_today_uncached`（后台）。
- 新增 `timeline/scheduler.rs`：启动 12s 后首次跑，之后每 90s 补全今日未缓存片段；与 `timeline_describe_lock` 共用防并发。
- `timeline::list_uncached_for_date`：按日列出无有效 cache 的 segment 上下文。
- `lib.rs setup`：调用 `timeline::scheduler::spawn`。

### 前端

- `TimelineView` 仍仅在 active 时拉取展示；打开页时直接读已写入的 `timeline_ai_cache`，仅补当前页缺口。

2. 最大化 Cursor 等窗口时桌宠应仍可见；仅游戏/F11 真全屏时隐藏。
3. `npm run tauri:dev` 下桌宠应在 dev server 就绪后出现（最多约 20s）。
