# 23 · 供应商 JSON 配置与适配器工厂架构

**日期**：2026-07-02  
**类型**：架构重构

## 需求

- 供应商改为 JSON 配置文件导入
- 修改供应商不影响整个系统（无需改 Rust 硬编码）
- 使用适配器工厂等设计模式

## 架构

```
config/vendors.json          ← 内置供应商目录（可嵌入）
%AppData%/config/vendors.json ← 用户副本（优先加载，可自定义）

SQLite ai_config             ← 仅用户选择：密钥绑定、模型、选中项
```

### 模块

| 模块 | 职责 |
|------|------|
| `ai/catalog.rs` | 加载/seed JSON 供应商目录 |
| `ai/adapters/mod.rs` | 适配器工厂 `adapter_for()` |
| `ai/adapters/ollama.rs` | Ollama 适配器 |
| `ai/adapters/openai.rs` | OpenAI 兼容 + ping 测试策略 |
| `ai/config.rs` | 用户状态与目录合并为 IPC `AiConfig` |
| `ai/providers.rs` | 测试/拉取模型（委托工厂） |
| `ai/adapter.rs` | chat_text / chat_vision 入口 |

### JSON 字段

- `adapter`: `ollama` | `openai`（工厂路由）
- `test.strategy`: `list_models` | `openai_or_ping`
- `hints`: 错误与空列表提示（火山 Plan 等）

### 迁移

- 旧版 SQLite 内嵌 `vendors` 数组 → 提取为 `vendor_vault` 映射
- `removed_vendor_ids` 在 JSON 中声明废弃项（如 `volcano_ark`）

### 前端

- 设置页展示 `vendors.json` 路径
- IPC: `app_get_vendors_config_path`
