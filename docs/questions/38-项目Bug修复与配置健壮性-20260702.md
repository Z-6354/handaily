# 38 · 项目 Bug 修复与配置健壮性

**日期**：2026-07-02  
**类型**：Bug 修复 / 健壮性

## 背景

开发环境曾出现人设工坊 `Send` 编译错误（持 DB 锁跨 await）；另排查出 AI 配置保存、思考模型迁移、JSON 解析等隐患。

## 修复项

### 1. 人设工坊 AI 命令（已修复）

- `character_preprocess` / `merge_text` / `generate_skill`：`prepare` 短锁 → `run_async` → 再锁写库，避免 `MutexGuard` 跨 await 导致无法编译/运行

### 2. AI 配置保存误清空 imported_models

- **现象**：设置页只改模型下拉时，若 payload 未带 `imported_models`，后端 serde 默认为 `[]`，会清空已导入模型
- **修复**：
  - 后端 `AiConfig::save`：incoming 为空时保留 DB 已有 `imported_models`
  - 前端 `normalizeAiConfig` + `saveAi` 合并完整配置

### 3. 思考模型旧配置迁移

- 无 `thinking_*` 的旧用户：迁移时 `thinking_vendor_id` 跟随 `text_vendor_id`
- `apply_vendor_defaults`：无思考默认模型时回退到 `text_model`

### 4. 角色 JSON 解析容错

- AI 返回带前言/markdown 时，从文本中提取 `{...}` 再解析

### 5. 人设工坊 DB 锁

- 角色 CRUD IPC 改用 `lock_conn`，支持 poison 恢复
