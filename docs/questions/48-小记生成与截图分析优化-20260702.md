# 48 · 小记生成与截图分析优化

**日期**：2026-07-02  
**类型**：报告 / 截图 / 桌宠计划  
**问答条数**：1

## 问答一

### 问题

根据桌宠集成计划，优化小记等文档生成逻辑；优化截图提示词和返回结果解析。

### 答复

#### 小记 / 报告（`report/mod.rs`）

1. **时间线 AI 缓存优先**：`collect_timeline_lines` 读取 `timeline_ai_cache`，格式 `- 10:00–10:25 【开发】简介`；无缓存时 fallback 到 `activity_key` 标签。
2. **时段小记补数据**：`period-summary` 在时段小结为空时，自动附带时间线片段给 AI。
3. **AI 失败不阻断**：`report_generate` 网络/解析失败时降级本地 Markdown。
4. **Markdown 清洗**：`sanitize_report_markdown` 去除 ``` 包裹。

#### 截图（`vision-screenshot.md` + `ai/adapter.rs`）

1. 提示词注入 `type_list`、`activity_hint`（项目/页面级活动内容）。
2. 新增 `ai/json_util.rs` 统一 JSON 提取。
3. `normalize_vision_category` 将 development/browsing 等映射到用户工作类型。
4. `confidence` 限制 0.05～0.99；`summary` 截断 80 字。
5. 视觉 fallback 使用 `activity_label` 而非泛化文案。

#### 桌宠计划更新

`handaily_桌宠集成_4be8ad40.plan.md` 新增 Phase 1.5 与 todos：`doc-gen-optimization`（已完成）、`vision-prompt-parse`（已完成）、`pet-remark-data-source`（待做：气泡复用 timeline/insight 数据）。
