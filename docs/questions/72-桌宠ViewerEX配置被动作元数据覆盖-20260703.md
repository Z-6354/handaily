# 桌宠 ViewerEX 配置被动作元数据覆盖导致部件散落

**日期**: 2026-07-03  
**主题**: 选择随机动作后 Spine 部件散落、未使用 `.config.json`

## 现象

用户曾用 Live2D ViewerEX 生成 `.config.json`（type=9）解决 Spine 部件散落。选择随机动作后问题复现，模型回到「全部部件散落」的初始错误状态。

日志中无 Spine 专用报错；主要可见 vision AI 配额错误（无关）。

## 根因

1. **`save_animation_meta` 写入 `model.json`**：保存随机/点击/待机动作时，把 `animations`、`random_animations` 等字段合并进 `model.json`。若该文件本身是 ViewerEX Spine 配置（或用户仅在此文件存配置），会污染配置结构。
2. **`find_config_file` 未识别 `model.json` 中的 type=9**：部分用户只有 `model.json` 作为 ViewerEX 配置，此前不会被当作 config 加载，回退到目录内任意 `.skel`，导致散落。
3. **刷新动作时强制 `playIdle()`**：`petRefreshAnimations` 切换待机未重置 setup pose，随机/一次性动作后骨骼状态异常。

## 修复

### Rust `models.rs`

- 动作元数据改存 **`animations.meta.json`**，不再写入 `model.json`
- `find_config_file` 增加：若 `model.json` 为 ViewerEX（type=9 + skeleton），也作为 config 源
- `read_model_name` / `write_model_name`：ViewerEX 格式的 `model.json` 用 `display_name.txt` 存显示名

### 前端 `spinePet.ts`

- 加载与切回待机前：`clearTracks` + `setToSetupPose` + `setSlotsToSetupPose`
- `configureAnimations({ soft: true })`：设置页仅更新随机表时不打断当前姿态

### 前端 `main.ts`

- `refreshPetAnimations` 使用 soft 模式

## 验证

1. 模型目录保留 ViewerEX 的 `.config.json`（或 type=9 的 `model.json`）
2. 勾选随机动作 → 部件不散落，配置仍指向正确 skel/atlas
3. 随机动作播放结束后恢复正常待机姿态

## 若已污染

若 `model.json` 已被混入动作字段但仍含 `type:9` 与 `skeleton`，现会自动识别为 config；动作数据在 DB 与 `animations.meta.json` 中。
