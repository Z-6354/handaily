# 83 — 切换模型碎块回归：回退热重载

**日期**: 2026-07-04  
**主题**: doc 82 同窗口热重载导致碎块复发

## 现象

doc 82 将 `set_active_model` 改为 `nudge_pet` 同窗口热重载以加速后，切换模型再次出现碎块；仅「关闭桌宠再启用」可恢复。

## 根因

同 canvas 上 dispose/重建 Spine（`skipBoot` 快路径跳过 `finalizePetAssembly`、缩短 assemble pump）在 Windows WebView 上仍不可靠；与 doc 80 结论一致。**可靠路径必须是销毁并重建桌宠 WebView**。

## 修复

1. **恢复** `restart_pet_window`：`set_active_model` 重新走 destroy → wait close → `show_pet`
2. **撤销** `initSpine` 快路径分支，统一走 `syncAnimations` + `finalizePetAssembly`
3. **保留** doc 82 中不引发碎块的优化：
   - `pet_read_model_bundle` 批量读资源
   - 切换前 Rust 侧预热文件页缓存
   - 前端 `preloadModelAssets`、窗口尺寸未变时跳过 resize

## 权衡

切换模型仍会有短暂窗口闪烁（WebView 重建），但应无碎块。速度略慢于热重载，但优于 doc 82 之前无预热的重建路径。
