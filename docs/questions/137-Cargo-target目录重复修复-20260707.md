# 137 · Cargo target 目录重复修复

**日期**：2026-07-07

## 现象

`HANDAILY/target/debug/deps` 与 `src-tauri/target/debug/` 两套目录，磁盘占用翻倍。

## 原因

- 在**仓库根**执行 `cargo build --manifest-path src-tauri/Cargo.toml` → 产物写入 `HANDAILY/target/`
- 在 **`src-tauri/`** 内执行 `cargo run --bin …` → 产物写入 `src-tauri/target/`
- 两套各自含 `debug/` + `debug/deps/`（Cargo 正常子目录，不是 bug）

## 修复

`.cargo/config.toml`：

```toml
[build]
target-dir = "src-tauri/target"
```

无论从哪执行 cargo，统一写入 `src-tauri/target/`。

## 清理与用法

```powershell
# 删除误生成的根目录 target/
.\scripts\clean-stale-target.ps1

# 推荐：从仓库根用 npm 包装命令
npm run check:rust
npm run live2d:import -- --plan mcp/blhx-wiki/plan.json
npm run roster:check
```

## 说明

`debug/deps/` 存放依赖中间文件，`debug/*.exe` 是可执行文件——**同一套 target 内的正常结构**，勿删 `deps/`。
