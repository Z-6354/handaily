# 启动时主窗口先小后大

**日期**：2026-07-04  
**分类**：应用壳层 / 窗口

## 现象

启动时主窗口先以较小尺寸（约 960×640）出现，再放大到铺满工作区，有闪动。

## 根因

1. **`tauri.conf.json`** 默认 `width: 960`、`height: 640`。
2. **`tauri.preview.conf.json`**（`npm run tauri:dev` 使用）将主窗口覆盖为 `"visible": true`，窗口在 `setup()` 完成**之前**就以小尺寸绘制到屏幕上。
3. **`lib.rs` setup** 里再调用 `maximize()`，Windows 播放最大化动画 → 用户看到「小窗 → 大窗」。

Release 默认 `visible: false` 时问题较轻；开发模式最明显。

## 修复

1. **`tauri.preview.conf.json`**：`visible` 改回 `false`，与正式配置一致；由 setup 在准备好尺寸后再 `show()`。
2. **`lib.rs`**：新增 `prepare_main_window_for_first_show`——窗口仍隐藏时，用当前显示器 `work_area` 的 position + size 直接铺满（无最大化动画）；失败时回退 `maximize()`。开发模式再 `show()` + `set_focus()`。

从托盘再次打开主窗口时仍保持用户上次大小，不重复强制铺满（见 Q14）。

## 验证

`npm run tauri:dev` 启动后，主窗口应直接以工作区大小出现，无先小后大的动画。
