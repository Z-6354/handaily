# 134 · 人物删除级联与列表性能优化

**日期**：2026-07-07

## 需求

866 人物 + 1400+ 模型导入后：

1. 删除人物/皮肤时遗留 manifest 与磁盘模型
2. 人物列表分页、搜索仍全量重建，占用高

## 实现

### 级联删除

| 操作 | 行为 |
|------|------|
| 删除人设（人物页） | `persona_delete` → 同步 `purge_character_for_persona`：移除 `characters/manifest` 条目；删除无其它引用的用户模型；活跃人物切回默认 |
| 删除皮肤（详情 · 皮肤卡片 ×） | IPC `characters_remove_skin`：manifest 移除皮肤；可选删 `pet-models` 目录；若删的是当前皮肤则切回默认皮肤并刷新桌宠 |

内置人物/内置模型不可删。

### 性能

- `list_model_id_set`：人物列表只扫模型目录名，不 inspect Spine
- `resolve_all_characters`：persona 查找改 HashMap，O(n²) → O(n)
- `list_characters_page`：先筛选/分页再构建 `CharacterBrief`，不再每页重建 866 条
- 搜索/收藏：`characters_list_page` 支持 `query`、`favoritesOnly`、`favoriteIds`，前端不再 `loadAllBrief` 拉全量

### 前端

- `useCharacterRoster({ query, favoritesOnly, favoriteIds })` + 250ms 防抖
- `CharacterSkinPicker` 皮肤卡片悬停显示删除
- 删除确认文案更新（含模型文件说明）
