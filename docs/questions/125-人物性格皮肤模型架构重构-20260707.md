# 125 · 人物 / 性格 / 皮肤 / 模型架构重构

**日期**：2026-07-07  
**标签**：架构、人物、persona、桌宠、皮肤、模型

## 问题

将原本的「模型 + 性格」双平行线架构，改为：**人物** → 含 **性格**（persona）+ 多个 **皮肤**（skin）→ 每个皮肤对应一个 **Spine 模型**（model）。

## 结论速览

| 层级 | 职责 | 存储 |
|------|------|------|
| **人物 Character** | 统一入口：切换时同步性格 + 皮肤 + 模型 | `characters/manifest.json` + SQLite `active_character_id` / `active_skin_id` |
| **性格 Persona** | AI system prompt、Skill、结构化 profile | `personas/`（不变） |
| **皮肤 Skin** | 外观变体，指向 model_id | manifest 内 `skins[]` |
| **模型 Model** | Spine 资源、动作、台词库 | `public/assets/pet/` + 用户 `pet-models/` |

内置映射示例：人物 `cheshire`（柴郡）→ 性格 `cheshire` → 皮肤 `default` → 模型 `chaijun`。

## 后端

### 新增

- `characters/manifest.json` — 内置 5 人物及皮肤 → 模型映射
- `hanpet/src-tauri/src/character/mod.rs` — `list/get/set_active/set_skin`、迁移、双向 sync

### IPC（已有注册，现可编译运行）

- `characters_list` / `characters_get_detail`
- `characters_set_active` — 切换人物 → 默认皮肤 + persona + model
- `characters_set_skin` — 切换皮肤 → model（性格随人物）

### 兼容

- `persona_set_active` → `character::sync_from_persona`
- `pet_set_model` → `character::sync_from_model`
- 启动 `migrate_on_startup`：由旧 `active_persona_id` + `pet_model_id` 推导人物/皮肤
- 导入模型 `pet_commit_import` → 为当前人物追加皮肤

## 数据流

```
characters_set_active(id)
  → active_character_id + default skin
  → persona_set_active(persona_id)
  → pet_set_model(skin.model_id)

characters_set_skin(character_id, skin_id)
  → active_skin_id
  → pet_set_model(skin.model_id)
```
