# 桌宠弹出气泡 Bug 修复

## 问题

时间线 AI 保存后桌宠气泡不显示，或调度器气泡也不弹出。

## 根因

1. **全屏误判**：`is_foreground_fullscreen()` 用窗口矩形与显示器 `rcMonitor` 比较，最大化 Cursor/浏览器（尤其任务栏自动隐藏时）会被当成全屏，桌宠被 `hide`，气泡事件发到隐藏窗口用户看不到。
2. **emit 时未恢复桌宠**：`emit_remark` 未在因全屏抑制而隐藏时临时 `show_pet`。
3. **前端竞态**：`listen("pet-remark")` 未 `await` 注册，启动瞬间事件可能丢失；窗口 `hidden` 期间气泡计时结束，恢复可见时无内容。

## 修复

### Rust

- `win32::is_foreground_fullscreen`：排除 `SW_SHOWMAXIMIZED` 与带 `WS_CAPTION` 的窗口，仅判断沉浸式全屏。
- `emit_remark(app, st, ...)`：桌宠启用且因全屏抑制隐藏时，时间线等 remark 前临时恢复显示；并 `ensure_pet_window`。

### 前端

- `setupPetEvents()`：`await listen` 后再 `reloadPet`。
- `showBubble`：先移除 `visible` 强制重绘；`document.hidden` 时缓存 `pendingBubble`，可见后补弹。
- `.pet-bubble`：`z-index: 10`、`word-break` 防遮挡与溢出。
