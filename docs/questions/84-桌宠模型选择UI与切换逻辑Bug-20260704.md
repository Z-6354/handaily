# 84 — 桌宠模型选择 UI 与切换逻辑 Bug

**日期**: 2026-07-04  
**主题**: 优化模型选择 UI；修复切换/导入状态不同步

## UI 优化

- 新增 `PetModelPicker`：卡片网格（概览页）+ 头部紧凑下拉
- 当前模型高亮、「内置 / 导入」徽章、切换中 spinner
- 切换时 Toast 提示「正在加载…」

## 发现的 Bug

1. **切换模型后 UI 显示已启用但后端未启用**  
   `switchModel` 在 `refreshStatus` 后执行 `setPetEnabled(true)`，未调用 `petSetEnabled`，仅改本地 state。

2. **导入模型后同样问题**  
   `runCommitImport` 仅 `setPetEnabled(true)`，未走 IPC。

3. **乐观更新 modelId**  
   切换失败时需手动回滚；现由 `refreshStatus` 统一以服务端为准。

4. **全局 `busy` 阻塞**  
   模型切换改用独立 `switchingModelId`，避免与导入等操作混用同一锁（切换不再占 `busy`）。

## 修复

- `switchModel`：去掉错误 `setPetEnabled`；`switchingModelId` + loading/success 反馈
- `runCommitImport`：若未启用则 `petSetEnabled(true)` 后再 `refreshStatus`
- 移除原生 `<select>`，改用 `PetModelPicker`
