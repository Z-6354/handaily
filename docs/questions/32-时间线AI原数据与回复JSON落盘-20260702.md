# 32 · 时间线 AI 原数据与回复 JSON 落盘

**日期**：2026-07-02  
**类型**：新功能

## 需求

工作时间线调用 AI 时，将发送给模型的**原数据**与**模型回复**一并存入 JSON 文件。

## 实现

### 存储路径

`%AppData%/xiaohan-daily/timeline-ai/{YYYY-MM-DD}/{HHMMSS.mmm}-off{offset}-n{batch}.json`

启动时自动创建 `timeline-ai/` 目录。

### JSON 结构（每批 ≤20 条片段）

```json
{
  "created_at": "...",
  "date": "2026-07-02",
  "limit": 50,
  "offset": 0,
  "batch_size": 3,
  "request": {
    "vendor_id": "...",
    "model": "...",
    "persona_id": "cheshire",
    "system_prompt": "人设全文…",
    "user_prompt": "渲染后的 timeline-segment-describe…",
    "work_type_options": ["开发", "…"],
    "segments": [ { "id", "app_name", "window_title", "insights", … } ]
  },
  "response": {
    "used_ai": true,
    "raw": "模型原文",
    "parsed": [ { "id", "work_type", "summary" } ],
    "error": null
  },
  "finalized": [ { "started_at", "work_type", "summary", "used_ai" } ]
}
```

- **不写入 API Key**
- 未配置模型时仍落盘，`response.error` 说明走本地回退
- 已缓存片段不会重复调 AI，故不产生新 JSON

### 代码

- `timeline/json_log.rs`：`save_batch_log`
- `timeline_describe` IPC：每批生成后写文件
- 时间线页底部显示日志目录路径
