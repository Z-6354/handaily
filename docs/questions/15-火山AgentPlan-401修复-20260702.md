# 15 · 火山 Agent Plan API 401 修复

**日期**：2026-07-02  
**类型**：Bug 修复

## 问题

使用火山引擎 **Agent Plan** API Key 测试时报 HTTP 401，请求 URL 为 `/api/v3/models`。

## 根因

Agent Plan 与按量计费 Ark **网关不同、密钥不通用**：

| 套餐 | Base URL（OpenAI 兼容） | 密钥来源 |
|------|-------------------------|----------|
| Agent Plan | `https://ark.cn-beijing.volces.com/api/plan/v3` | Agent Plan 控制台专用 Key |
| Coding Plan | `https://ark.cn-beijing.volces.com/api/coding/v3` | Coding Plan 控制台 |
| 按量计费 | `https://ark.cn-beijing.volces.com/api/v3` | API Key 管理 |

原「火山引擎 Agent」误配为 `/api/v3`，Agent Plan 密钥在此网关无效 → 401。

## 修复

- `volcano` 供应商改为 **Agent Plan** 网关 `/api/plan/v3`
- 新增 `volcano_ark`（按量）、`volcano_coding`（Coding Plan）
- 加载配置时自动迁移旧 `/api/v3` → `/api/plan/v3`
- 测试/导入：优先 `/models`，Plan 套餐 fallback `GET /ping` + 内置 `ark-code-latest` 等
- API Key 使用前 `trim()` 去空白
- 401 提示说明密钥与网关须匹配

## 用户操作

1. 确认密钥来自方舟控制台 **Agent Plan** 页面（非按量 API Key 管理）
2. 设置 → AI 配置 → 选「火山引擎 Agent Plan」
3. 文本模型推荐 `ark-code-latest`
4. 重新测试 / 导入
