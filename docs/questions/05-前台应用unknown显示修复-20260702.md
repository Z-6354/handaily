# 05 — 前台应用出现 unknown 的修复

**日期**：2026-07-02  
**标签**：tracker、前台采集、显示名、聚合

## 问题

今日看板「应用排行」中出现 `unknown` 条目，时长被计入前台使用。

## 根因

1. **`poller.rs`**：`get_foreground_snapshot()` 返回 `None` 时（本应用在前台、无 HWND 等），用 `app_name: "unknown"` 构造占位快照，且若用户非空闲会写入非 idle 段。
2. **`win32.rs`**：`exe_stem` 在路径解析失败时回退为字面量 `"unknown"`。
3. **`derive_aggregation_key`**：exe 与 app_name 皆空时返回 `"unknown"`。
4. **`display_name::friendly_from_key`**：未将 `"unknown"` 映射为「未知应用」。
5. 部分进程 `QueryFullProcessImageNameW` 失败（提权/UWP），导致 exe 路径为空。

## 修复

| 模块 | 改动 |
|------|------|
| `poller.rs` | 有前台快照才处理；无前台且空闲 → idle 占位段；无前台且活跃 → 跳过 tick（延续上一段） |
| `win32.rs` | `GetProcessImageFileNameW` 回退；标题解析应用名；`exe_stem` 不再返回 `unknown` |
| `title_parse.rs` | 新增：从窗口标题 `"文档 - App"` 解析名；`is_ignored_agg_key` |
| `mod.rs` | 聚合键增加 `window_title` 回退；空键 → `__idle__`；扩展 UWP `WindowsApps` |
| `display_name.rs` | 支持 `window_title`；`unknown`/`__idle__` → 「未知应用」 |
| `stats.rs` | 聚合时跳过 ignored 键（历史 unknown 不进排行） |

## 验证

```bash
cd hanpet/src-tauri && cargo test
```

重启开发服务后观察应用排行，不应再出现 `unknown`；本应用获得焦点时段应归入上一应用或 idle。
