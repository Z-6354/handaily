# 桌宠点击动作锁与拖动预览

**日期**: 2026-07-03  
**主题**: 点击动作串行、拖动位置预览

## 需求

1. **点击锁**：单击触发的 Spine 动作播放期间，再次点击无效；动作完成后才可继续点击。拖动不受影响。
2. **拖动预览**：拖动桌宠窗口时显示虚线框与提示，实时跟随鼠标并 clamp 到桌面内。

## 实现

### 点击锁（`hanpet/src/pet/spinePet.ts`）

- `clickActionBusy` 标志：`playTap()` 开始时置 true，动画 `complete` 回调置 false。
- `handleClick(): boolean` — 若 busy 则返回 `false`，不触发动作。
- `isClickActionBusy()` 供外部查询。

随机动作、AI 气泡触发的 `playAnimation` 不占用点击锁，仅用户单击触发的 tap/click 动作串行。

### 拖动预览（`hanpet/src/pet/main.ts` + `pet.css`）

- 弃用 `startDragging()`，改为 `screenX/Y` 差值 + `setPosition` 手动拖动，与区域 clamp 一致。
- 超过 `DRAG_THRESHOLD` 后进入拖动：`pet-drag-preview` 虚线框 +「拖动中 · 松开保存」提示；角色半透明 + 阴影。
- `requestAnimationFrame` 节流位置更新；松手后 `savePosition()` 持久化。

## 验证

1. 快速连点桌宠 — 仅第一次播放点击动作，其余忽略。
2. 动作播放中拖动 — 仍可拖走窗口。
3. 拖动时出现虚线框与提示，松手后位置保存且不越界。
