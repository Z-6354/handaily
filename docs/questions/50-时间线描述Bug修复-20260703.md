# 50 · 时间线描述 Bug 修复

**日期**：2026-07-03  
**类型**：时间线 / AI 简介 Bug 修复

## 问题

时间线卡片上的「描述/简介」仍可能出现机器格式（如「开发：xxx · 窗口「…」」），或未按人设生成 AI 口吻，而是显示整段 window_title。

## 原因

1. **AI id 对不上**：提示词示例用了 `"id":"0"`，模型常返回序号 id，与后端 `started_at` 不一致 → 解析失败 → 降级本地简介或直接显示 window_title
2. **机器格式 cache 漏过滤**：`plan_describe` 会过滤旧 cache 触发重生成，但 `timeline_describe` 最终 `get_cached` 未过滤，仍可能把旧机器格式返回前端
3. **AI 返回机器式 summary 未拦截**：`parse_ai_map` / `finalize_chunk` 未校验 summary 是否机器格式
4. **前端 fallback**：无 AI 条目时 `fallbackText` 会原样展示长 window_title

## 修复

| 模块 | 改动 |
|------|------|
| `timeline/mod.rs` | `parse_ai_map(raw, chunk)` 支持序号 id 回退；拒绝机器 summary；`filter_valid_cache`；`finalize_chunk` 校验 AI 结果 |
| `ipc/commands.rs` | 最终返回前 `filter_valid_cache` |
| `timeline-segment-describe.md` | 示例 id 改为 started_at 格式，禁止 0/1/2 |
| `TimelineView.tsx` | fallback 识别机器式 window_title，改用人话占位 |

## 验证

```powershell
cd src-tauri && cargo test timeline::tests
cd .. && npm run build
```

刷新时间线页；旧机器格式 cache 会被视为无效并重新生成。
