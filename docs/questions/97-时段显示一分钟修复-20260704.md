# 97 · 时段显示只有一分钟的修复

**日期**：2026-07-04  
**标签**：时段记录、时间线、checkpoint、热力图

## 问题

时间线、时段记录热力图等界面里，每条记录几乎都只显示约 **1 分钟**，与真实连续使用时长不符。

## 根因

`tracker/writer.rs` 的 `checkpoint()` 原先每 60 秒把内存中的 **open segment** 落盘并立即开启「续接段」，导致 DB 里充满 ~60s 的碎片行。时间线虽会合并相邻同应用段，但 activity_key 若随窗口标题微调变化则无法合并；热力图 `segment_count` 也会虚高。

## 修复

1. **`checkpoint()`**：仅 flush `pending` 短片段 + `PRAGMA wal_checkpoint(PASSIVE)`，**不再拆分 open segment**。
2. **`flush_all_segments()`**：退出时计算 open 段 `duration_ms` 后一次性 `flush_segment`。
3. **`audio_monitor`**：移除 60s 音频 checkpoint 拆分，会话结束/退出时再落盘。
4. **`TodayHeatmap`**：去掉 `|| 1` 强制显示 1 分钟的逻辑。

## 行为变化

- 正常运行：同一次前台使用在切换应用/空闲/退出前只写 **一条** segment，时长连续。
- 崩溃恢复：最多丢失当前未落盘的 open 段（启动时 `recover_orphan_segments` 仍闭合历史孤儿行）。
