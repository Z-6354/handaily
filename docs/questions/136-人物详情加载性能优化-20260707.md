# 136 · 人物详情加载性能优化

**日期**：2026-07-07

## 现象

点击人物详情需等待数秒甚至更久。

## 根因

`get_character_detail` 存在三重开销：

1. `resolve_all_characters()` — 每次读 866 人 manifest + persona + 扫 1486 模型目录（~400ms）
2. 再次 `load_manifest()`
3. **`skin_infos` → `model_name` → `list_models()`** — 对每个皮肤全量 inspect 1486 个 Spine 目录（最慢，可达数秒×皮肤数）

## 修复

### Rust

- `find_character_meta`：只查单人 manifest 条目，不再构建全量 roster
- `model_display_name` / `model_names_for_ids`：只读 `display_name.txt`，不 inspect Spine
- `list_characters` 改用轻量 `model_names_map`
- `set_active_character` / `set_active_skin` 同样走单人查询

### 前端

- 详情 **内存缓存**（再次打开同人物即时显示）
- 点击卡片先用列表 **brief 占位**（名称/简介立即展示），后台拉完整 Skill/皮肤

## 验证

```powershell
cd src-tauri
cargo run --bin roster_check
# detail 应在数十 ms 级（视 skill_md 文件大小）
```
