# 审计剩余项继续优化（第三轮）

## 问题

继续处理审计剩余项：M5 共享 runtime、时段 AI 日志、时间线服务端筛选、后台 worker 优雅退出。

## 修复摘要

### M5 — 共享 Tokio runtime
- 新增 `ai/runtime.rs`：`OnceLock` 持有 current-thread runtime
- `PreparedTextChat::run_sync`、`chat_vision`、`vision::execute_prepared` 统一 `runtime::block_on`

### 时段 AI 失败日志
- `period_scheduler::run_period_job` 与 `period::analyze_period`：AI 调用失败 `eprintln`，再回退规则引擎

### 时间线服务端时间筛选
- `stats::filter_segments_since` + `query_timeline(..., since_minutes)`
- `plan_describe`、`stats_timeline`、`timeline_cached`、`timeline_describe` 支持 `since_minutes`
- 前端切换 30m/1h/2h 时传参并重载，移除纯客户端过滤

### 后台 worker 优雅退出
- 分析/时段 worker 改用 `recv_timeout(1s)` + `stop_flag` 检测
- `AnalysisCoordinator::join_worker`、`PeriodScheduler::join_all`
- 进程 `Exit` 时 `AppState::join_ai_workers()`
